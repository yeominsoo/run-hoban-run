## 구현된 기능

- 최대 800명의 참가자 입력
- 3가지 트랙 선택
- 60초 기준 레이스 시뮬레이션
- 랜덤 장애물 10회 발생
- 장애물 참가자별 통과 확률 49%
- 참가자별 5% 확률로 10초 스퍼트 스킬 발동
- 지정 순위 범위 통과 판정
- 기본 예시 범위 `55등`부터 `155등`
- 동일 시드 기반 결과 재현
- 전체 순위 CSV 내보내기
- Docker 기반 테스트 실행
- AI 작업 문서 세트

## 게임 룰 요약

- 참가자는 줄 단위로 입력하며 빈 줄은 무시합니다.
- 참가자 수가 800명을 초과하면 엔진에서 오류를 냅니다.
- 트랙별로 속도 편차, 장애물 손실, 스킬 배수가 다릅니다.
- 장애물은 경기 중 10회 무작위 시점에 발생합니다.
- 장애물 실패 시 위치 손실과 감속이 적용됩니다.
- 스킬이 발동된 참가자는 10초 동안 트랙별 속도 배수를 받습니다.
- 최종 60초 시점 위치가 높은 순서로 순위를 계산합니다.
- 지정한 시작/종료 순위 안에 들어온 참가자만 통과자로 표시합니다.

상세 규칙은 `.ai-work/01-game-rules.md`와 `docs/game-design.md`를 기준으로 확인합니다.

## UI/UX 구조

- 화면은 레이스 트랙을 넓게 보여주는 가로 중심 레이아웃입니다.
- 좌우 고정 패널은 제거했고, `설정`과 `결과`는 레이어 패널로 열립니다.
- 상단 조작 바에는 `설정`, `결과`, `규칙 갱신`, `레이스 시작`, `새 레이스`, `CSV`가 있습니다.
- 시드 입력은 사용자에게 노출하지 않고 hidden 값으로 유지합니다.
- 참가자별 고유 색상을 부여해 순위가 바뀌어도 참가자를 추적하기 쉽게 했습니다.
- 움직이는 참가자 표시는 이름이 왼쪽, 말 이미지가 오른쪽에 배치됩니다.
- 트랙에는 호수, 나무 라인, 관중석, 출발선, 결승선 그래픽이 있습니다.
- 레이스 종료 시 결과 레이어가 자동으로 열립니다.

## 실행과 테스트

로컬 실행:

```bash
npm run start
```

접속 주소:

```text
http://localhost:5174
```

Node 테스트:

```bash
npm test
```

Docker 테스트:

```bash
npm run test:docker
```

마지막 확인 이력:

- `npm test`: 6개 테스트 통과
- `npm run test:docker`: Docker 컨테이너 안에서 6개 테스트 통과
- `node --check src/app.js`: 통과
- `node --check src/raceEngine.js`: 통과
- `git diff --check`: 통과

## 운영 시나리오

1. `npm run start`로 서버를 엽니다.
2. 브라우저에서 `http://localhost:5174`에 접속합니다.
3. 상단 `설정`을 눌러 참가자 목록과 통과 순위 범위를 입력합니다.
4. 필요한 트랙과 재생 속도를 선택합니다.
5. `규칙 갱신`으로 현재 입력값을 시뮬레이션에 반영합니다.
6. `레이스 시작`을 눌러 경기를 진행합니다.
7. 종료 후 자동으로 열린 `결과` 레이어에서 통과자를 확인합니다.
8. 필요하면 `CSV`로 전체 순위를 내보냅니다.

## 주의할 점

- 현재 앱은 별도 백엔드나 DB 없이 정적 파일로 동작합니다.
- 참가자 명단 저장 기능은 없습니다. 새로고침 전 필요한 결과는 CSV로 내려받아야 합니다.
- 시드는 UI에 표시하지 않지만 내부 값으로 사용됩니다.
- 같은 참가자 목록, 같은 트랙, 같은 통과 범위, 같은 시드라면 같은 결과가 재현됩니다.
- 실제 행사 화면 비율이 넓은 디스플레이인지 확인한 뒤 레인 수와 글자 크기를 추가 조정할 수 있습니다.
- GitHub 푸시는 환경 인증 상태에 의존합니다. 현재 로컬 브랜치는 원격 `feature/race-lottery-game`을 추적합니다.
