## 기본 정보

- 게임 제목: `달려라 검단호수공원 호반써밋`
- 장르: 경마형 추첨 웹게임
- 경기 시간: 시뮬레이션 기준 60초
- 구현 기준 파일: `src/raceEngine.js`

## 참가자 규칙

- 참가자는 최소 1명 이상 필요합니다.
- 참가자는 최대 800명까지 입력할 수 있습니다.
- 참가자 입력은 줄 단위 이름 목록으로 처리합니다.
- 빈 줄은 무시합니다.
- 내부 참가자 ID는 입력 순서 기준으로 1부터 부여합니다.

관련 구현:

- `MAX_PARTICIPANTS = 800`
- `normalizeParticipants()`
- `createSampleParticipants()`

## 트랙 규칙

트랙은 3가지 유형입니다.

| ID | 이름 | 특징 |
| --- | --- | --- |
| `lake` | 호수 순환 트랙 | 균형 잡힌 속도와 안정적인 흐름 |
| `hill` | 언덕 스퍼트 트랙 | 빠른 상위권과 큰 장애물 손실 |
| `forest` | 숲길 변수 트랙 | 변수가 크고 회복 기회가 많은 코스 |

트랙별로 속도 계수, 변동성, 장애물 손실, 스킬 배수가 다릅니다.

관련 구현:

- `TRACK_TYPES`
- `getTrack()`

## 장애물 규칙

- 경기 중 랜덤 장애물이 총 10회 발생합니다.
- 장애물 발생 시점은 60초 안에서 무작위로 생성되고 시간순으로 정렬됩니다.
- 장애물 통과 확률은 참가자별 49%입니다.
- 장애물을 통과하면 소폭 전진 보너스를 받습니다.
- 장애물에 실패하면 위치 손실과 감속 시간이 적용됩니다.

관련 구현:

- `OBSTACLE_COUNT = 10`
- `OBSTACLE_PASS_PROBABILITY = 0.49`
- `generateObstacleSchedule()`
- `applyObstacle()`

## 스킬 규칙

- 참가자별로 5% 확률로 스퍼트 스킬이 한 번 발동합니다.
- 스킬은 발동 후 10초 동안 유지됩니다.
- 스킬이 활성화된 동안 트랙별 스킬 배수에 따라 속도가 증가합니다.

관련 구현:

- `SKILL_TRIGGER_PROBABILITY = 0.05`
- `SKILL_DURATION_SECONDS = 10`
- `createRacerState()`
- `advanceRacers()`

## 순위 및 통과 규칙

- 최종 60초 시점의 누적 위치가 높은 순서로 순위를 정합니다.
- 위치가 같으면 참가자 ID가 낮은 순서가 앞섭니다.
- 운영자가 지정한 순위 범위 안에 들어온 참가자만 통과합니다.
- 예시: `55등`부터 `155등`까지 지정하면 총 101명이 통과합니다.

관련 구현:

- `normalizePassRange()`
- `simulateRace()`
- `ranking`
- `passers`

## 재현성 규칙

- 시드가 같고 참가자 목록과 설정이 같으면 같은 결과를 재현합니다.
- 시드를 비우면 실행 시점 기준으로 새 시드가 생성됩니다.

관련 구현:

- `createSeed()`
- `createSeededRandom()`

## 작업 종료 핸드오프 규칙

- 개발 작업을 마칠 때는 `.ai-work/HANDOFF.md`를 생성하거나 최신 상태로 갱신합니다.
- 핸드오프 문서에는 현재 브랜치, 커밋, 서버 상태, 구현 범위, 실행 방법, 테스트 결과, 남은 리스크, 다음 작업 제안을 포함합니다.
- 코드 변경이 없더라도 운영 방식, 배포 상태, 테스트 결과가 바뀌면 핸드오프 문서를 갱신합니다.
- 핸드오프 문서 갱신 후 `.ai-work/README.md` 목차에서 해당 문서 링크가 유지되는지 확인합니다.
