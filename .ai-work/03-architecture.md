## 전체 구조

앱은 엔진과 UI를 분리한 구조입니다.

- `src/raceEngine.js`: 입력값을 받아 레이스 결과를 계산하는 순수 로직
- `src/app.js`: DOM 이벤트와 화면 렌더링을 담당하는 브라우저 로직
- `index.html`: 운영 패널, 레이스 화면, 결과 패널의 기본 마크업
- `styles.css`: 화면 스타일과 반응형 레이아웃

## 시뮬레이션 엔진

핵심 함수는 `simulateRace()`입니다.

입력:

- 참가자 이름 목록
- 트랙 ID
- 통과 시작 순위
- 통과 종료 순위
- 시드

출력:

- 레이스 제목
- 트랙 정보
- 통과 범위
- 장애물 이벤트 목록
- 시간별 프레임
- 최종 순위
- 통과자 목록
- 요약 통계

엔진은 UI에 의존하지 않기 때문에 Node 테스트에서 직접 검증할 수 있습니다.

## 브라우저 UI

`src/app.js`는 다음 역할을 담당합니다.

- 참가자 텍스트 입력 처리
- 80명, 180명, 800명 샘플 생성
- 트랙 선택 버튼 렌더링
- 통과 순위 범위 입력 처리
- 시드 생성과 재사용
- 재생 속도 제어
- 레이스 프레임 애니메이션
- 실시간 순위와 최종 결과 표시
- CSV 내보내기

## 데이터 흐름

1. 운영자가 참가자, 트랙, 통과 범위, 시드를 입력합니다.
2. `prepareRace()`가 `simulateRace()`를 호출합니다.
3. 엔진이 60초 레이스 전체 프레임과 최종 결과를 생성합니다.
4. UI가 0초 프레임을 먼저 표시합니다.
5. `startRace()`가 재생 속도에 맞춰 프레임을 순서대로 렌더링합니다.
6. 60초 지점에 도달하면 최종 통과자와 순위표를 표시합니다.

## 결과 데이터

최종 순위 항목에는 다음 값이 포함됩니다.

| 필드 | 의미 |
| --- | --- |
| `rank` | 최종 순위 |
| `name` | 참가자 이름 |
| `distance` | 최종 누적 거리 |
| `obstaclePasses` | 장애물 통과 횟수 |
| `obstacleFails` | 장애물 실패 횟수 |
| `skillTriggered` | 스킬 발동 여부 |
| `skillStart` | 스킬 시작 시각 |
| `skillEnd` | 스킬 종료 시각 |
| `passed` | 지정 순위 범위 통과 여부 |
