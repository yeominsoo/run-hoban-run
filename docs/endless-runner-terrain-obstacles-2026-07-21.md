# 안엘런 지형·장애물 그래픽 개선 (2026-07-21)

## 적용 결과

- 지면 기준을 화면 높이의 `72% → 82%`로 내려 바닥 점유 영역을 `28% → 18%`로 줄였다.
- 하늘 그라데이션, 구름, 태양, 2중 원경 언덕과 장식용 공중섬을 패럴랙스로 구성했다.
- 바닥은 장애물과 같은 소프트 3D 화풍의 반복형 잔디·흙 단면 텍스처로 교체했다. 잔디층,
  뿌리, 지층, 자갈, 암석과 공동이 함께 이동하며 기존 잔디 잎과 꽃 장식을 위에 유지한다.
- 구덩이는 초원 지형이 끊긴 절벽과 어두운 암벽 균열로 표현한다.
- 프로덕션 장애물은 그루터기, 이끼 바위, 가시밭, 공중 잔디 지형, 벌, 파랑새,
  줄을 타고 내려온 뱀 7종이다.
- 같은 지면 텍스처 위에 월드 좌표로 고정된 초원·야생화·바위 구간을 이어 붙였다. 구간별
  풀 높이, 꽃 밀도, 지면 색조와 표면 돌이 달라지지만 모두 지형과 같은 속도로 이동한다.

특정 상용 게임의 캐릭터·블록·파이프·코인·지형을 복제하지 않고 밝고 읽기 쉬운 횡스크롤
플랫포머의 공간감만 참고한 오리지널 그래픽이다.

## 라운드별 난이도

| 라운드 | 신규 요소 | 기본 회피 |
|---|---|---|
| 1 | 기본 장애물과 공중 발판 | 점프, 발판 착지, 슬라이드 |
| 2 | 구덩이와 지형 단절 | 점프 |
| 3 | 더 빠르게 다가오는 벌·파랑새 | 슬라이드 |
| 4 | 아래 구덩이와 함께 상하 이동하는 발판 | 발판 높이에 맞춘 점프 |
| 5 | 지상에서는 안전하지만 점프 궤도와 겹치는 비행몹 | 지상 유지 또는 체공 조절 |
| 6 | 1단 점프 최고점보다 높은 3단 바위탑 | 2단 점프 |
| 7 | 이동 발판 비중 증가 | 연속 착지 |
| 8 | 바위탑과 높은 지형 비중 증가 | 2단 점프 |
| 9 | 점프 높이 비행몹 비중 증가 | 지상 유지·급강하 |
| 10 | 2단 점프 바위탑 연속 | 2단 점프 |
| 11 | 세 고급 패턴 정밀 혼합 | 패턴별 판단 |
| 12 | 최고 속도의 마스터 마라톤 | 종합 |
| 13 이상 | 이동 발판 → 점프 높이 비행몹 → 바위탑을 라운드마다 순환 보장 | 종합 |

15초마다 실제 라운드 번호가 계속 오르며, 1~12라운드는 서로 다른 속도·간격·구덩이·공중
장애물·특수 패턴 비율을 가진다. 13라운드부터는 12단계의 안전 상한을 유지하면서 첫 특수
패턴을 세 종류로 순환해 같은 코스가 반복되는 느낌을 줄인다. 장애물 최소 간격은 단계별로
감소하되 `300px` 아래로 줄지 않으며, 최고 속도도 약 `506px/s`에서 더 가속하지 않는다.
코인은 생성 뒤 위치를
바꾸지 않고 모든 장애물과 최소 안전거리를 둔 달리기·점프 높이에만 놓는다. 미래에 생성될
비행 생물이 화면 안에서 코인을 따라잡지 못하도록 접근속도 기반 여유 거리도 함께 확보한다.

공중 잔디 지형은 위에서 내려올 때 발 위치를 윗면에 고정하는 착지형 발판이다. 발판이 화면
왼쪽으로 빠져나가면 캐릭터는 자연스럽게 낙하하며, 옆면·아랫면 접촉은 사망 판정에서 제외한다.
4라운드부터 등장하는 이동 발판은 같은 착지 규칙을 사용하되 상하로 움직이며 바로 아래에
발판 폭만큼의 구덩이가 함께 놓인다. 모바일의 낮은 프레임률에서도 착지가 누락되지 않도록
일반 발판보다 넓은 판정 여유를 두고, 초기 이동 폭과 속도는 낮게 시작해 후반 단계에서만
점진적으로 증가한다.

2단 점프 바위탑은 높이 `154px`로, 1단 점프 이론상 최고 높이 약 `131px`보다 높고 2단 점프의
최대 도달 범위 안에 있다. 점프 높이 비행몹은 지면에서 달릴 때는 머리 위로 지나가지만 일반
점프의 정점 부근에서는 충돌하도록 지면에서 `112px` 떨어진 높이에 배치한다.

벌·파랑새의 충돌 하단은 달리는 캐릭터 머리와 겹치고 슬라이드 캐릭터 위에는 여유가 생기게
설정한다. 비행 생물은 지형 스크롤에 더해 벌 `60~76px/s`, 파랑새 `87~102px/s`의 라운드별
접근 속도로 플레이어 쪽을 향해 날아온다. 12단계 확장 뒤에는 일반·점프 높이 패턴을 합쳐
벌은 최대 `110px/s`, 파랑새는 최대 `128px/s`로 제한한다. 작은 상하 움직임, 날갯짓,
진행 반대쪽 잔상으로
이동 방향을 읽을 수 있게 하되 슬라이드 안전 통로는 유지한다.

줄을 타고 내려온 뱀은 새·벌과 달리 지형에 고정된 슬라이드 전용 장애물이다. 줄은 화면 위로
이어지고 몸만 약하게 흔들리며, 가로 위치에는 추가 접근속도를 주지 않는다. 충돌 하단은 달리는
캐릭터와 겹치되 슬라이드 캐릭터 머리 위에는 여유가 남도록 맞췄다.

최종 프롬프트는 현재 소프트 3D 캐릭터 시트를 렌더링 화풍 참조로 사용하고, 2열×3행에
그루터기·가시밭·공중 잔디 지형·벌·파랑새·이끼 바위를 하나씩 배치하도록 작성했다. 각 항목은
작은 모바일 크기에서도 읽히는 굵은 실루엣, 높은 국부 대비, 재질 디테일을 요구했다. 배경은
제거 가능한 평면 `#00ff00`으로 제한하고 특정 게임 프랜차이즈를 연상시키는 파이프·블록·코인·
버섯·성 등의 요소를 금지했다.

이미지 생성기가 만든 미세한 크로마 배경 편차는 공식 제거 도구로 투명화하고, 빌더가 각 셀과
별도 뱀 원본을 256×256 RGBA로 정규화한다. 원본 아틀라스는 정확한 `#00ff00`, 초록 뱀과
겹치지 않는 뱀 원본 배경은 자홍색 계열로 보존한다.

지면 최종 프롬프트는 장애물 아틀라스를 화풍 참조로만 사용하고, 화면 전체를 채우는 정측면
초원 단면을 요청했다. 윗부분 10~14%의 잔디층과 아래의 흙 지층·뿌리·돌·작은 공동을 요구하고,
하늘·원근·분리된 발판·캐릭터·동물·텍스트·프랜차이즈 요소는 금지했다. 빌더는 좌우 56px을
대칭 블렌딩해 양 끝 픽셀을 일치시키므로 스크롤 반복 경계가 드러나지 않는다.

## 빌드와 검증

```bash

python3 tools/verify_endless_runner_terrain_assets.py
```

에셋 프리로드, 지면 비율, 장애물 카탈로그, 12단계 프로필과 13라운드 이후 순환, 이동 발판
치수·이동량, 1단/2단 점프 도달 높이, 점프 높이 비행몹 판정, 코인 안전 동선, 충돌 종료,
2단 점프와 캐릭터 액션 동기화다.
