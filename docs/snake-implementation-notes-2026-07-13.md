# 스네이크 비틀기 구현 노트 (2026-07-13)

## 원본 스펙 (v1)

- 뱀이 먹이를 먹으며 성장, 자기 몸에 부딪히면 게임 오버
- 비틀기: 벽 없음(반대편으로 통과), 시간 갈수록 속도 업, 속도 업 구간마다 색깔 테마 변경
- 입력: 방향키/WASD/스와이프(모바일)
- Canvas 사용, 모바일 주의사항: 스와이프 방향 감지 필요

## 결정과 이유

1. **격자 크기**: 16×22 고정 논리 격자, 셀 크기는 `min(stageWidth/16, stageHeight/22)`로
   계산해 어떤 화면 비율에서도 격자가 중앙에 레터박스로 들어가도록 했다. 세로로 긴 모바일
   화면(16:22 ≈ 4:5.5)을 기준값으로 잡아, 데스크톱 가로 화면에서도 좌우 여백만 생기고
   격자 형태 자체는 always 동일하게 유지된다.
2. **속도 증가 주기**: 15초마다 틱 간격을 ×0.88배(최소 90ms까지)로 단축. v1이 "시간 갈수록
   속도 업"이라고만 하고 구체 주기가 없어서, 볼 피하기의 30초 주기보다 짧게 잡았다 — 스네이크
   한 판이 보통 볼 피하기보다 밀도 높게(먹이 몇 개만 먹어도 몸이 길어져 난이도가 이미 오르는
   구조) 짧게 끝나는 경향이 있어, 같은 30초 주기면 게임이 끝나기 전에 속도 변화를 거의 못 볼
   수 있다고 판단했다.
3. **색깔 테마 전환**: 속도가 오를 때마다(15초마다) 5가지 파스텔 팔레트를 순환한다. 팔레트는
   본 저장소의 `pastel-theme.css` 톤에서 색상만 다르게 뽑았다(핑크·민트·라벤더·골드·스카이
   블루) — 완전히 새로운 색 체계를 만들지 않고 기존 팔레트 언어 안에서 변주했다.
4. **반대 방향 입력 처리**: 현재 진행 방향과 정반대인 입력(예: 오른쪽으로 가는 중 왼쪽 입력)은
   무시한다. 이건 스펙에 없지만 이 장르의 보편적 규칙이다 — 반대 방향을 그대로 받아들이면
   몸통에 바로 부딪혀 사실상 "즉사 버튼"이 되어버려 정상적인 게임 플레이가 성립하지 않는다.
5. **틱 기반 이동**: 매 프레임 부드럽게 움직이는 대신 고정 틱(속도에 따라 90~220ms)마다 한
   칸씩 이동하는 고전 스네이크 방식을 그대로 따랐다. v1 스펙이 "격자 기반" 여부를 명시하진
   않았지만, 스네이크 장르 자체가 격자+틱 이동이 표준이라 별도로 명시할 필요가 없다고 봤다.
6. **점수**: 먹이를 먹은 횟수를 점수로 사용한다(시작 몸길이 3칸은 점수에 포함하지 않음).
