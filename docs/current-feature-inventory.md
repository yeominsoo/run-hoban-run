# 현재 기능 인벤토리

작성 기준: React/TypeScript 전환 전 정적 JavaScript 구현.

## 저장소와 패키지 매니저

- 루트 앱 파일: `index.html`
- 스타일: `styles.css`
- 브라우저 제어: `src/app.js`
- 게임 규칙 엔진: `src/raceEngine.js`
- 3D 렌더러: `src/threeRaceScene.js`
- 테스트: `tests/raceEngine.test.js`
- lock 파일: `package-lock.json`
- 패키지 매니저: npm

## 기존 게임 기능

| 항목           | 현재 구현                                       | 보존 기준                                   |
| -------------- | ----------------------------------------------- | ------------------------------------------- |
| 시작 버튼      | 상단 `레이스 시작` 버튼이 `startRace()` 호출    | React 버튼에서도 동일하게 레이스 재생 시작  |
| 말 목록        | textarea 줄 단위 참가자 입력, 빈 줄 무시        | 최대 800명, 줄 단위 입력 유지               |
| 샘플 생성      | 80명, 180명, 800명 버튼                         | 동일 샘플 생성 유지                         |
| 주행 로직      | `simulateRace()`가 0.5초 단위 프레임 계산       | 물리엔진 없이 TypeScript 엔진에서 직접 계산 |
| 속도/랜덤 요소 | 시드 기반 RNG, 트랙별 속도/편차/스킬 배수       | 같은 입력과 시드는 재현 가능해야 함         |
| 결승 판정      | 제한 시간 최종 위치 기준 순위 산정              | 최종 순위와 통과 판정 유지                  |
| 순위 표시      | 실시간 상위 10명, 트랙 상위 8명, 최종 상위 40명 | React 컴포넌트로 동일 정보 표시             |
| 리셋           | `새 레이스`가 새 시드 생성 후 준비              | 동일 동작 유지                              |
| 규칙 갱신      | 현재 참가자/트랙/통과 범위로 레이스 재생성      | 동일 동작 유지                              |
| 트랙 선택      | 호수/언덕/숲길 3가지                            | 동일 트랙과 설명 유지                       |
| 장애물         | 현재 비활성화                                   | 주행 결과를 물리엔진에 맡기지 않음          |
| 스킬           | 참가자별 5%, 10초 지속                          | 스킬 상태와 3D 줌인 트리거 유지             |
| 결과           | 통과자 목록, 통계, CSV 내보내기                 | 동일 데이터 유지                            |
| UI 문구        | `달려라 검단호수공원 호반써밋`, 한국어 운영 UI  | 주요 문구 유지                              |
| 이미지/사운드  | `assets/horse.svg`만 존재, 사운드 없음          | 사운드 도입 없음                            |

## 보존해야 할 핵심 규칙

- 참가자 수는 1명 이상 800명 이하.
- 레이스는 시뮬레이션 시간 기준 84초.
- 트랙은 3가지.
- 장애물은 현재 비활성화.
- 헬기 탈락 이벤트는 조별 3회 등장한다.
- 스킬 발동 확률은 참가자별 5%.
- 스킬 지속 시간은 10초.
- 통과자는 운영자가 지정한 최종 순위 범위로 결정.
- 같은 참가자 목록, 트랙, 통과 범위, 시드는 같은 결과를 재현.

## TypeScript 모듈 분해 대상

| 기존 단위                                             | 전환 대상                  |
| ----------------------------------------------------- | -------------------------- |
| `src/raceEngine.js`                                   | `src/game/raceEngine.ts`   |
| `src/app.js` 전역 상태                                | `src/stores/raceStore.ts`  |
| DOM 이벤트 바인딩                                     | React 이벤트 핸들러        |
| `renderLanes`, `renderLiveRank`, `renderFinalResults` | React 컴포넌트 렌더링      |
| `requestAnimationFrame` 재생 루프                     | React effect + store 상태  |
| `src/threeRaceScene.js`                               | React Three Fiber 컴포넌트 |
| CSV 생성                                              | TypeScript 엔진 유틸 유지  |

## 전환 시 추가할 확장 지점

- 헬기 탈락 이벤트는 별도 이벤트 모델로 관리한다.
- 타격 대상은 사격 직전까지 UI에 노출하지 않는다.
- 기수는 말 그룹 하위에 붙여 말 이동과 같은 transform을 따른다.
- 3D 렌더러는 프레임 데이터와 이벤트 상태만 소비하고 주행 결과를 계산하지 않는다.
- 물리엔진은 주행 계산이 아니라 추후 소품/센서/충돌 이펙트 전용으로만 연결한다.
