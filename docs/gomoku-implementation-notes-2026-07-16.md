# 오목(gomoku) 구현 노트 (2026-07-16)

리버시를 추가한 직후 사용자가 "오목도 있으면 좋겠다"고 이어서 요청해 추가한 게임.
리버시와 마찬가지로 계획서의 텐텐오락실 오마주 계열과 무관한 독립 요청이라 별도 Phase
구분 없이 바로 진행했다.

## 리버시 골격을 그대로 재사용

로비 대기 없이 join 즉시 3초 카운트다운 후 시작하는 1:1 자동 매칭, 자기 차례 30초
타임아웃(시간 초과 시 서버가 무작위 빈 칸에 대신 착수), 재접속(`RECONNECT_GRACE_MS`
45초) 등 서버·클라이언트 코드 구조를 리버시(`ws-server/reversi.mjs`,
`src/pages/reversi/main.ts`)에서 거의 그대로 가져왔다. 다른 점은 "돌을 뒤집는" 로직이
없고(오목은 한번 놓은 돌이 바뀌지 않음) 승리 조건이 "5개 연속"으로 바뀐 것뿐이다.

## 자유형(Freestyle) 오목으로 결정 — 렌주 룰 미구현

오목의 정식 대회 규칙(렌주 룰)은 흑에게 삼삼(3-3)·사사(4-4)·장목(6개 이상 연속) 금수를
적용해 선공 유리함을 상쇄한다. 이 저장소는 다음 이유로 **자유형(금수 없음, 5개 이상
연속이면 무조건 승리)**을 택했다:

- 렌주의 금수 판정(특히 삼삼 판정)은 규칙 자체가 꽤 복잡해서, 캐주얼 파티게임 모음에
  들어가는 미니게임치고는 구현·검증 비용 대비 가치가 낮다고 판단했다.
- 이미 리버시도 정식 대회 규칙(공식 오프닝 룰 등)까지 따지지 않고 표준 규칙의 핵심만
  구현한 전례가 있다 — 같은 기준을 오목에도 적용했다.
- 장목(6개 이상 연속)도 렌주처럼 흑에게만 무효 처리하지 않고 그냥 승리로 인정한다.

## 15×15 표준 크기, 30초 턴 타임아웃

보드 크기는 오목의 표준인 15×15를 그대로 썼다(19×19 바둑판을 오목에 쓰는 변형도 있지만
표준은 15×15). 턴 타임아웃은 리버시와 동일하게 30초로 맞췄다 — 둘 다 "생각할 시간이
필요한 턴제 전략 게임"이라는 성격이 같기 때문.

## 서버 권위 원칙

리버시와 동일하게 비밀 정보가 없는 완전 정보 게임이다. 클라이언트는 `place {row, col}`만
보내고, 승리 판정(방금 둔 자리에서 4방향 각각 양쪽으로 뻗어나가며 같은 색이 몇 개
연속되는지 계산해 5 이상이면 승리)은 항상 서버(`isWinningMove`/`longestLineThrough`)가
확정한다. 클라이언트는 이미 채워진 칸을 다시 클릭하지 못하게만 막고, 실제 규칙 판정은
하지 않는다.

## 검증

- Playwright UI 테스트(모바일 세로 390×844)로 호스트(흑)가 방을 만들고 게스트(백)가
  참가 → 자동 매칭 → 카운트다운 → 보드 화면까지 확인, 가로 한 줄에 5개를 연달아 두면
  승리 배너가 정확히 뜨는지 확인했다.
- raw WebSocket 스크립트로 5번째 돌을 놓기 **전까지는** `game_over`가 오지 않고, 5번째
  돌을 놓는 순간에만 `game_over`가 오는지(4개까지는 승리 처리되지 않는 경계값) 확인했다.

## 배포

WAS 배포는 앞선 게임들과 동일한 절차(`ws-server/README.md`)를 따른다.
