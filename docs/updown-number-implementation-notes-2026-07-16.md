# 업다운 넘버(updown-number) 구현 노트 (2026-07-16)

텐텐오락실 오마주 후보 10종(Phase 4) 중 세 번째 게임. 두더지 사냥·순서 기억 챌린지에 이어
"동시 랭킹형" 패턴 두 번째 적용 사례 — 문제 생성/판정 로직만 바뀌고 로비·재접속·카운트다운
골격은 그대로 재사용했다(계획서에서 예고한 대로).

## 스펙에 없어 직접 정한 수치와 근거

계획서 한 줄 명세("전원이 같은 숨은 정답 숫자를 업/다운 힌트로 동시에 추론, 가장 적은
시도로 먼저 맞힌 사람이 1등")만으로는 구체적인 범위·시간·랭킹 기준이 없어 아래를 확정했다.

- **숫자 범위 1~100**: 이진탐색으로 최대 7번이면 맞힐 수 있는 익숙한 범위. "업다운 게임"
  하면 보편적으로 떠올리는 규모라 별도 설명 없이도 감이 온다.
- **라운드 60초**: 두더지 사냥(반응속도, 30초)과 달리 매 시도마다 힌트를 읽고 다음 수를
  생각할 시간이 필요한 "추론" 장르라 더 길게 잡았다.
- **랭킹 기준 = 맞힌 순서(먼저 맞힌 사람이 1등), 시도 횟수는 부가 정보**: 명세 문구
  "가장 적은 시도로 먼저 맞힌 사람이 1등"을 "시도 횟수로 랭킹을 매기고 동률이면 순서로
  타이브레이크"로 읽을 수도 있었지만, 그러면 라운드 종료 전까지 누가 이기고 있는지 아무도
  볼 수 없어 재미가 없다. 대신 **맞힌 순간 그 자리에서 순위가 확정**(1등, 2등, ...)되도록
  해서 실시간으로 등수가 갈리는 긴장감을 살렸다. 아직 못 맞힌 사람들은 라운드 종료 시점
  마지막 추측이 정답에 가장 가까운 순으로 정렬해 "아깝게 놓쳤다"는 피드백을 준다.
- **전원 정답 시 조기 종료**: 시간이 남아도 연결된 전원이 맞히면 굳이 60초를 다 채울
  이유가 없어 즉시 라운드를 끝낸다.

## 서버 권위 원칙 적용

정답 숫자(`secret`)는 `game_over` 브로드캐스트 전까지 그 어떤 메시지에도 필드로 만들지
않는다 — 값을 숨기는 게 아니라 필드 자체를 안 만드는 이 저장소의 원칙을 그대로 따랐다.
힌트는 `up`/`down`/`correct` 문자열로만 전송되고, 그마저도 **추측한 본인에게만** 사설로
보낸다. 다른 참가자에게 브로드캐스트되는 `progress_update`에는 시도 횟수와 정답 여부만
있고 실제로 어떤 숫자를 불렀는지·힌트가 뭐였는지는 아예 없다 — 안 그러면 다른 사람의
추측과 힌트를 엿봐서 정답을 간접 유추할 수 있게 된다.

## 순서 기억 챌린지에서 발견한 "카운트다운 없는 시작" 버그를 여기서는 처음부터 반영

이 작업을 시작하기 직전, 순서 기억 챌린지가 "게임 시작" 클릭 직후 카운트다운 없이 곧바로
라운드가 진행되어 처음 하는 사람이 화면 전환 도중 룰도 못 읽고 첫 타일을 놓치는 문제를
사용자가 실제 플레이 중 발견했다(자세한 내용은
`docs/ws-lobby-tip-and-countdown-2026-07-16.md`). 그래서 업다운 넘버는 처음부터 두더지
사냥과 동일한 3초 카운트다운(`COUNTDOWN_MS`)과 로비의 "게임 방법" 팁 카드
(`src/shared/ws-lobby-tip.css`의 `.how-to-play`)를 갖추고 시작했다.

## 검증

- **raw WebSocket 스크립트**: 호스트+게스트 2개 소켓으로 방 생성→참가→시작→카운트다운→
  라운드 시작→둘 다 이진탐색으로 정답을 맞히는 전체 시퀀스를 왕복 확인. 두 클라이언트가
  받는 모든 프레임의 raw JSON 문자열에 `"secret"` 키가 `game_over` 이전에는 전혀 없음을
  직접 검사해 안티치트를 확인했다. 전원이 맞히자 60초를 기다리지 않고 즉시 `game_over`가
  온 것도 확인.
- **Playwright UI 테스트**: 모바일 세로(390×844) 뷰포트에서 로비 → 카운트다운 → 플레이
  화면까지 실제 UI로 이진탐색 추측을 입력해 UP/DOWN/정답 힌트가 올바르게 표시되고, 라이브
  보드에 시도 횟수가 갱신되는지 스크린샷으로 확인. 겹침 없이 정상 렌더링됨.

## 배포

WAS 배포는 두더지 사냥·순서 기억 챌린지와 동일한 절차(`ws-server/README.md` 참고)를
따른다 — `ws-server/nginx.conf`(저장소 사본)와 `~/rps-tls/nginx.conf`(WAS 실제 사본) 양쪽에
`/updown-number` location 블록 추가 후 `docker restart rps-tls` 필요.
