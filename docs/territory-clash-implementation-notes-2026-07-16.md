# 영역 쟁탈전(territory-clash) 구현 노트 (2026-07-16)

텐텐오락실 오마주 후보 10종 중 아홉 번째. Phase 5 "실시간 조작 동기화형" 계열의 두 번째
게임으로, 줄다리기 배틀에서 확립한 "1:1 자동 매칭(로비 대기 없이 join 즉시 시작) + 서버가
유일한 판정 권위를 갖는 실시간 상태" 골격을 그대로 재사용했다. 차이는 판정 대상이
스칼라 하나(줄다리기의 위치)에서 **격자 칸별 소유권**(48개의 개별 상태)으로 늘어난 것뿐이다.

## 스펙에 없어 직접 정한 수치와 근거

- **8행×6열(48칸)**: 합이 10 퍼즐(8×10, 드래그 선택 게임이라 칸이 많아도 됨)과 달리 이
  게임은 **개별 탭**이 핵심이라 모바일에서 오탭 없이 누르기 좋은 크기를 우선해 더 작게
  잡았다.
- **제한시간 30초**: 줄다리기 배틀과 동일하게 맞춰 Phase 5 세 게임이 "짧고 굵은 1:1
  대결"이라는 톤을 공유하도록 했다.
- **칸 재탈환 허용, 쿨다운 없음**: "영역 쟁탈"이라는 이름에 맞게 한 번 뺏긴 칸도 계속
  되찾을 수 있어야 진짜 쟁탈전이 된다고 판단했다 — 점령 후 일정 시간 보호(쿨다운)를 주는
  방식도 고려했지만, 그러면 후반에 스코어가 고착되어 뒤집기가 어려워지므로 캐주얼한
  파티게임 톤에는 맞지 않는다고 봤다.

## 서버 권위 원칙 — "경합 칸은 서버에 먼저 도착한 탭이 이긴다"

같은 칸을 두 사람이 거의 동시에 탭하면 어느 쪽 메시지가 서버에 먼저 도착하느냐로
승자가 갈린다 — 할리갈리 구현 때 확립한 "서버 수신 순서 기반 판정" 원칙을 그대로
따른다. 클라이언트는 탭 즉시 자기 색으로 낙관적으로 칠하지만(반응성 확보), 서버의
`cell_claimed` 브로드캐스트가 도착하면 그 값으로 무조건 덮어써 최종 확정한다 — 만약
내가 낙관적으로 칠했는데 상대가 먼저 도착했다면 브로드캐스트가 다시 상대 색으로
되돌린다. 이 "낙관적 갱신 후 서버 확정으로 덮어쓰기"는 줄다리기 배틀에서 의도적으로
쓰지 않은 패턴이다(그쪽은 위치가 매 프레임 비교되는 값이라 어긋나면 화면이 심하게
튈 수 있어 처음부터 서버 응답만 반영했다) — 반면 이 게임은 칸 단위라 어긋나도 그 칸
하나만 잠깐 다른 색으로 보였다가 바로 정정되는 정도라 낙관적 업데이트를 허용해도
안전하다고 판단했다.

## 검증

- **raw WebSocket 스크립트**: 호스트+게스트 2개 소켓으로 자동 매칭 → 카운트다운 →
  매치 시작까지 확인 후, 같은 칸을 "호스트 점령 → 게스트 탈환 → 호스트 재탈환" 순으로
  제출해 매번 `cell_claimed`의 `side`와 점수가 정확히 뒤바뀌는지 확인했다. 첫 시도에서
  소켓 큐에 이전 브로드캐스트가 남아있어 로그가 혼란스럽게 보인 적이 있었는데, 각 단계마다
  큐를 완전히 비우고 다시 검증한 결과 서버 로직 자체는 처음부터 정확했음을 재확인했다
  (테스트 스크립트의 큐 소비 순서 문제였을 뿐, 실제 버그는 아니었다). 자기 칸을 다시
  탭하면 아무 메시지도 오지 않는(no-op) 것도 확인.
- **Playwright UI 테스트**: 모바일 세로(390×844)에서 호스트가 방을 만들고 게스트가
  참가하면 자동 매칭 → 카운트다운 → 플레이 화면까지 진행해, 10개 칸을 연속 탭했을 때
  전부 호스트 색(파랑)으로 칠해지고 점수가 10으로 정확히 반영됨을 스크린샷으로 확인.
