## 이번 세션에서 새로 확립된 패턴 (다음에 비슷한 요청이 오면 참고)

### 1. WS 로비 "플레이팁 + 시작 버튼" 패턴

`docs/ws-lobby-tip-and-countdown-2026-07-16.md`에 상세 기록. 요지:
- 로비 패널의 `lobby-players` 아래, `게임 시작` 버튼 위에 `.how-to-play` 카드(공용 CSS
  `src/shared/ws-lobby-tip.css`)를 넣는다.
- "게임 시작" 클릭 직후 곧바로 게임을 시작하지 말고 3초 카운트다운(`COUNTDOWN_MS`,
  `.ws-countdown-number`)을 거친다 — 단, 이건 순서 기억 챌린지가 겪었던 특정 버그(첫
  라운드가 너무 빨리 지나가 버림)의 수정이라, **모든 WS 게임에 강제할 보편 규칙은
  아니다.** 실시간 동시입력 게임(mole-hunt류)에는 자연스럽게 맞지만, 알아서 속도를
  조절하며 진행하는 턴제 대화형 게임(라이어/마피아 등)에는 굳이 강제하지 않았다.

### 2. 선공/후공 결정전(가위바위보 + 팀 주사위) 패턴

`docs/starting-order-decider-2026-07-16.md`에 상세 기록. 요지:
- 판정 순수 로직은 `ws-server/starting-order.mjs`(`resolveRps`, `rollDiceOff`)에
  공용화돼 있다. 앞으로 "선공/후공이 있는" 새 게임을 만들 때는 이 모듈을 재사용할 것.
- 가위바위보 손 아이콘은 `src/shared/hand-icons.ts`(원래 `src/pages/rps/hand-icons.ts`
  였던 걸 이동)에 있다.
- 공용 CSS는 `src/shared/starting-order.css` (`.decide-*`, `.dice-off-*` 클래스).
- **판별 기준**: 1:1 또는 팀전이면서 "누가 먼저 두는가/움직이는가"가 승부에 영향을
  주는 턴제 게임만 대상. 동시입력 게임(두더지 사냥, 구구단 스퍼트 등)이나 매 프레임
  동시 조작 게임(줄다리기 배틀, 영역 쟁탈전)은 해당 없음 — 이미 판별해서 4개 게임에만
  적용 완료했으니, 새 턴제 대결 게임을 또 만들 때만 이 패턴을 검토하면 된다.

### 3. 새 WS 게임을 추가하는 절차는 변함없음

`docs/party-games-handoff-2026-07-03.md`에 있는 "새 WS 게임 추가 체크리스트"가 여전히
유효하다(모든 게임 서버는 `noServer: true` + `server.mjs`의 공용 upgrade 라우터 패턴).
1:1 자동매칭형(로비 대기 없이 join 즉시 카운트다운)과 그룹 로비형(host가 "게임 시작"
누름) 두 골격 모두 이제 예시가 여러 개 쌓여 있으니, 새 게임 성격에 맞는 쪽을 템플릿으로
그대로 복붙해서 시작할 것.

## 알아두면 좋은 것 (완료는 아니지만 확정된 판단)

- **QR/바코드 온보딩(계획서 Phase 6)**: `docs/game-collection-plan-v2-2026-07-13.md`에
  "미구현"으로 표시된 채 남아있다. 사용자가 명시적으로 요청하지 않는 한 먼저 시작하지
  말 것 — 이 세션에서도 건드리지 않았다.
- **타워 쌓기 턴제 멀티 모드**: 사용자가 잠깐 제안했다가 "턴제 쌓기는 하지 말자"고
  스스로 접었다(2026-07-16 대화 중). 다시 요청받기 전까지는 진행하지 않는다.
- **전략윷놀이 "빽도" 규칙**: 사용자가 확인해준 바로는 〈더 지니어스〉 원본 방영분에는
  없던 규칙이고, 이 저장소가 고전 윷놀이에서 참고해 추가한 것이다. 게임 자체에서
  빼자는 요청은 없었고, `docs/game-rules-comparison-index-2026-07-16.md`의 서술만
  "원작에 없는 의도적 확장"으로 정정했다 — 실제 게임 로직(`ws-server/strategy-yutnori-rules.mjs`의
  `frontCount === 1` → `backdo`)은 그대로 유지돼 있다.
- **나무위키 검증의 한계**: WebFetch로 나무위키 문서를 직접 열람하면 403이 떠서, 이
  세션의 모든 나무위키 인용은 WebSearch 요약을 통한 간접 확인이다. 더 정밀한 검증이
  필요하면 사람이 직접 문서를 열어 대조해야 한다(`game-rules-comparison-index` 문서
  하단에도 이 한계를 명시해뒀다).
