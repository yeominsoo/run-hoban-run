# 3D 웹게임 기획서

## 목표

`run-hoban-run`은 브라우저에서 실행되는 3D 레이스 연출형 웹게임으로 새로 제작한다.

기존 경마 추첨 게임 구현은 폐기하고, 3D 캐릭터 레이스, 관전 연출, 토너먼트 진행, 최종 우승자 발표를 중심으로 다시 설계한다.

## 레퍼런스 방향

`우마무스메`는 다음 관점에서만 참고한다.

- 캐릭터가 달리는 3D 레이스의 속도감
- 경기 전후 연출과 카메라 컷 전환
- 참가자별 개성과 능력치가 결과에 영향을 주는 구조
- 관전자가 현재 순위와 역전 흐름을 쉽게 이해하는 UI
- 최종 우승자 발표의 무대감

주의할 점:

- 우마무스메의 캐릭터, 세계관, 명칭, 의상, UI, 음원, 모델, 애니메이션을 복제하지 않는다.
- 자체 캐릭터 콘셉트와 자체 룰을 사용한다.
- 참고는 제품 방향성과 연출 품질 기준으로만 사용한다.

## 핵심 게임 콘셉트

- 장르: 3D 레이스 시뮬레이션 + 행사 추첨/토너먼트 게임
- 실행 환경: 데스크톱 브라우저 우선, 대형 현장 디스플레이 대응
- 플레이 방식: 운영자가 참가자 명단과 토너먼트 옵션을 입력하면 3D 레이스가 자동 진행
- 결과 방식: 조별 레이스를 거쳐 결승 참가자를 선발하고, 결승 상위 N명을 최종 우승자로 선정

## 1차 MVP 범위

1. 참가자 입력
   - 줄 단위 참가자 이름 입력
   - 최대 참가자 수 제한
   - 샘플 참가자 생성

2. 토너먼트 룰
   - 조당 인원 설정
   - 조별 진출 인원 설정
   - 최종 우승 인원 설정
   - 동일 시드 기반 결과 재현

3. 3D 레이스 화면
   - Three.js 기반 3D 트랙
   - 참가자별 3D 러너 모델
   - 레이스 진행에 따른 위치 이동
   - 출발선, 결승선, 관중석, 트랙 장식
   - 실시간 순위 표시

4. 카메라 연출
   - 전체 트랙을 보는 기본 카메라
   - 선두권 추적 카메라
   - 결승선 근접 카메라
   - 우승자 발표 카메라

5. 결과 화면
   - 토너먼트 라운드 요약
   - 결승 순위
   - 최종 우승자 목록
   - CSV 내보내기

## 3D 모델링 방향

초기 모델은 직접 제작 가능한 절차적 저폴리 스타일로 시작한다.

- 러너: 단순 사람형 또는 마스코트형 캐릭터
- 애니메이션: 팔/다리 왕복, 몸 기울기, 속도별 보폭 변화
- 개성 부여: 참가자별 색상, 등번호, 이름표
- 확장 계획: GLB 모델 교체 가능 구조

권장 구조:

- `public/models/`에 GLB/GLTF 모델 배치
- `src/scene/`에 Three.js 장면 구성
- `src/game/`에 레이스/토너먼트 엔진
- `src/ui/`에 설정/결과 패널

## 기술 스택 후보

- Vite
- TypeScript
- Three.js
- React 또는 순수 Three.js
- Vitest
- Playwright

초기 구현은 빠른 검증을 위해 `Vite + TypeScript + Three.js`를 우선 검토한다. React는 UI 복잡도가 커지는 시점에 도입 여부를 다시 판단한다.

## 레이스 룰 초안

- 각 참가자는 기본 속도, 가속, 체력, 막판 스퍼트 값을 가진다.
- 레이스는 고정 시간 시뮬레이션으로 계산한다.
- 화면 재생 속도는 연출 속도만 바꾸고 결과 자체는 시드로 고정한다.
- 조별 레이스 상위 N명이 다음 라운드에 진출한다.
- 결승 레이스 상위 N명이 최종 우승자가 된다.

## 화면 구성 초안

- 상단: 프로젝트명, 현재 라운드, 시간, 주요 조작 버튼
- 중앙: 풀스크린에 가까운 3D 레이스 장면
- 하단: 실시간 순위와 참가자 상태
- 설정 레이어: 참가자/토너먼트/시드/연출 옵션
- 결과 레이어: 토너먼트 진행표, 결승 순위, 최종 우승자

## 개발 순서

1. 빈 Vite 프로젝트 재구성
2. Three.js 렌더링 스파이크
3. 절차적 3D 러너 모델과 달리기 애니메이션 제작
4. 토너먼트 엔진 작성
5. 3D 레이스 진행과 엔진 결과 연결
6. 카메라 연출 추가
7. 설정/결과 UI 추가
8. 테스트와 Playwright 화면 검증 추가

## 검증 기준

- 브라우저에서 3D 장면이 빈 화면 없이 렌더링된다.
- 참가자가 실제로 트랙 위를 이동한다.
- 최소 데스크톱/모바일 뷰포트에서 UI가 겹치지 않는다.
- 같은 시드와 같은 설정이면 같은 최종 우승자가 나온다.
- 토너먼트 설정이 잘못되면 명확한 오류를 낸다.
- 최종 우승자 수가 설정값과 일치한다.
