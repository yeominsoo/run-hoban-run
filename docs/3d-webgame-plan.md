# 3D 웹게임 기획서

## 목표

`run-hoban-run`은 브라우저에서 실행되는 3D 레이스 연출형 웹게임으로 새로 제작한다.

기존 경마 추첨 게임 구현은 폐기하고, 다양한 색상과 무늬의 말이 달리는 3D 레이스, 관전 연출, 토너먼트 진행, 최종 우승자 발표를 중심으로 다시 설계한다.

## 레퍼런스 방향

`우마무스메`는 다음 관점에서만 참고한다. 단, 이 프로젝트의 주자는 사람이 아니라 말이다.

- 3D 레이스의 속도감과 카메라 박력
- 경기 전후 연출과 카메라 컷 전환
- 참가자별 개성과 말별 랜덤 속도 구간이 결과에 영향을 주는 구조
- 자동 레이스를 관전하며 순위 변화와 스킬 발동을 따라가는 구조
- 관전자가 현재 순위와 역전 흐름을 쉽게 이해하는 UI
- 최종 우승자 발표의 무대감

주의할 점:

- 우마무스메의 캐릭터, 세계관, 명칭, 의상, UI, 음원, 모델, 애니메이션을 복제하지 않는다.
- 사람 캐릭터가 직접 달리는 구조를 사용하지 않는다.
- 자체 캐릭터 콘셉트와 자체 룰을 사용한다.
- 참고는 제품 방향성과 연출 품질 기준으로만 사용한다.

## 핵심 게임 콘셉트

- 장르: 3D 레이스 시뮬레이션 + 행사 추첨/토너먼트 게임
- 실행 환경: 데스크톱 브라우저 우선, 대형 현장 디스플레이 대응
- 플레이 방식: 운영자가 참가자 명단과 토너먼트 옵션을 입력하면 3D 레이스가 자동 진행
- 결과 방식: 조별 레이스를 거쳐 결승 참가자를 선발하고, 결승 상위 N명을 최종 우승자로 선정
- 주자 콘셉트: 참가자별로 색상, 무늬, 번호가 다른 말
- 스킬 콘셉트: 말 위의 기수가 스킬 발동 순간 과장된 포즈와 짧은 퍼포먼스를 보여준다.
- 비주얼 스타일: 현실 비율보다 귀엽고 알아보기 쉬운 캐주얼 마스코트형 말
- 레이스 규모: 한 레이스는 최대 20마리까지 출전한다.
- 참가자 규모: 전체 참가자는 최대 500명까지 입력할 수 있고, 초과 인원은 받지 않는다.
- 진행 방식: 20마리를 넘는 참가자는 조별 토너먼트 라운드로 자동 편성한다.

## 우마무스메식 기본 룰 변환

원본 게임의 수치 공식, 고유 스킬, 캐릭터 시스템은 복제하지 않는다. 대신 기본 레이스 구조를 단순화해 자체 말 레이스 룰로 변환한다.

### 레이스 조건

- 레이스는 거리, 마장, 방향, 트랙 상태를 가진다.
- 거리 타입은 단거리, 마일, 중거리, 장거리로 나눈다.
- 마장은 잔디와 더트로 시작한다.
- 방향은 좌회전, 우회전으로 둔다.
- 트랙 상태는 양호, 다습, 불량 정도로 단순화한다.
- 거리와 트랙 상태는 레이스 표시와 시드 계산에 포함하되, 참가자별 속도/체력 특성 차이는 만들지 않는다.

### 공통 출발 조건

- 속도, 체력, 파워, 근성, 지능 같은 참가자별 능력치 특성은 사용하지 않는다.
- 마장 적성, 거리 적성 같은 참가자별 적성 등급도 사용하지 않는다.
- 모든 말은 같은 기본 완주 시간과 같은 출발 조건에서 시작한다.
- 출발 지연도 참가자별 능력치로 계산하지 않는다.
- 승부는 말마다 다르게 배정되는 20개 논리 주행 구간의 속도 배율과 헬리콥터 랜덤 탈락 이벤트로 갈린다.

### 구간별 랜덤 속도 변화

- 선두형, 선행형, 추격형, 막판형 같은 고정 작전 타입은 사용하지 않는다.
- 트랙을 시각적으로 속도 타일처럼 칠하지 않고, 말마다 20개 논리 구간의 랜덤 속도 배율을 가진다.
- 같은 구간에 있어도 말마다 배율이 달라지므로 선두 말이 감속하고 후방 말이 가속하면서 역전 상황이 생길 수 있다.
- 구간 배율도 시드, 라운드, 조, 말 ID를 조합해 계산하므로 같은 조건에서는 같은 레이스 흐름이 재현된다.

### 레이스 페이즈

- 출발: 모든 말이 같은 기준에서 출발한다.
- 초반/중반/종반: 말별 구간 배속에 따라 위치가 갈린다.
- 같은 말도 구간마다 감속, 유지, 가속, 폭주 상태가 달라질 수 있다.
- 선두 말이 감속 구간에 들어가고 후방 말이 가속 구간에 들어가면 역전이 발생한다.

### 스킬

- 장애물 횟수와 통과 확률 시스템은 사용하지 않는다.
- 스킬 기본 발생 확률은 3%로 고정한다.
- 스킬이 발동하면 5초 동안 기수 포즈와 이펙트가 표시된다.
- 스킬은 경기당 최대 1회만 발동한다.
- 스킬별로 발동 구간, 기수 포즈, 이펙트, 중계 문구에 차이를 둔다.
- 스킬 발동 중에는 기수가 물구나무, 눕기, 춤추기, 양팔 벌리기 같은 특수 포즈를 취한다.
- 스킬 발동 여부도 시드 기반으로 재현되지만 완주 시간에는 보정값을 주지 않는다.

### 경기별 숨김 랜덤 스킬 배정

- 참가자 입력이 확정되면 각 참가자에게 말 프로필을 자동 부여한다.
- 스킬은 참가자에게 영구 고정하지 않고, 매 경기 시작 시 출전 말마다 1개씩 숨김 배정한다.
- 참가자는 경기 전 자신이 어떤 스킬을 받았는지 알 수 없다.
- 스킬은 레이스 중 실제로 발동되는 순간 연출과 중계 문구로 처음 공개된다.
- 발동하지 않은 스킬은 경기 종료 후에도 공개하지 않는다.
- 다음 라운드 또는 다음 경기에서는 같은 참가자라도 다른 스킬을 받을 수 있다.
- 스킬 배정은 랜덤이지만 시드 기반으로 계산해 같은 참가자 명단, 같은 토너먼트 설정, 같은 시드에서는 경기별 스킬 배정이 재현된다.
- 경기별 스킬 시드는 `전체 시드 + 라운드 번호 + 조 번호 + 참가자 ID`를 조합해 만든다.
- MVP는 스킬 6개로 시작한다.
- 모든 스킬은 공통으로 `3% 발동 확률`, `5초 지속`, `경기당 최대 1회 발동` 규칙을 따른다.
- 스킬별 차이는 발동 가능 구간, 기수 포즈, 이펙트 색상, 중계 문구로 표현한다.
- 추후 확장 시 스킬 개수, 발동 조건, 희귀도를 늘릴 수 있다.

MVP 스킬 풀:

- 로켓 스타트: 출발/초반 구간에서 발동한다. 기수가 물구나무 포즈를 취하고 짧은 불꽃 이펙트를 표시한다.
- 코너 댄스: 코너 또는 중반 구간에서 발동한다. 기수가 춤추는 포즈를 취하고 리듬감 있는 별 이펙트를 표시한다.
- 드러눕기 질주: 중반 이후 발동한다. 기수가 말 위에 누운 포즈를 취하고 바람줄기 이펙트를 표시한다.
- 결승선 광란: 종반/라스트 스퍼트 구간에서 발동한다. 기수가 양팔을 벌리고 환호하는 포즈를 취한다.
- 진흙 튀기기: 더트 또는 불량 트랙에서 발동한다. 말 뒤쪽에 흙먼지 이펙트를 표시한다.
- 잔디 미끄럼: 잔디 또는 양호 트랙에서 발동한다. 말 발밑에 녹색 잔상 이펙트를 표시한다.

### 랜덤 헬리콥터 탈락 이벤트

- 헬리콥터 탈락 룰은 기존 기획서에는 없었고, 2026-05-22 구현 기준으로 신규 반영한다.
- 4명 이상이 출전하는 레이스에서는 헬리콥터가 시드 기반 랜덤 타깃을 들고 등장한다.
- 헬리콥터는 레이스 시작부터 노출하지 않고, 선두 말이 1/3 지점을 통과할 때 골인지점 쪽에서 등장한다.
- 헬리콥터가 등장하면 현재 순위와 무관하게 선두권부터 최하위권까지 전체 출전 말을 동일한 후보 풀로 두고, 중복 없이 출전자의 약 1/3을 랜덤 타깃으로 지정한다.
- 타깃이 된 말들은 선두 말이 1/2 지점을 통과하는 시점부터 순차적으로 탈락 처리되며, 각 탈락 사이에는 최소 연출 간격을 둬 화면이 과하게 산만해지지 않게 한다.
- 타깃으로 지정된 말들은 해당 레이스에서 진출/우승할 수 없다.
- 헬리콥터 이벤트도 전체 시드, 라운드, 조, 출전 수를 조합해 계산하므로 같은 조건에서는 같은 타깃과 등장 시점이 재현된다.
- 경기마다 다른 타깃을 보고 싶을 때는 UI의 `준비` 버튼으로 새 시드를 생성한다.
- 헬리콥터는 트랙 중간이 아니라 골인지점 쪽에서 나타나며, 골인지점 바깥에서 접근한 뒤 타깃 기수를 조준한다.
- 헬리콥터는 한 번 도착한 뒤에는 사라졌다가 재등장하지 않고, 골인지점 위치에서 타깃만 바꿔가며 연속 발사한다.
- 연출은 저격/타격을 과도하게 사실적으로 표현하지 않고, 헬리콥터 접근 카메라 컷, 하부 총구, 총구 섬광, 총알 궤적, 기수 피격 표시, 말 쓰러짐 수준의 캐주얼 게임 연출로 제한한다.
- 총알 발사 직전부터 명중 직후까지 전체 레이스는 순간 슬로우모션으로 느려지고, 카메라는 총알 궤적을 따라가며 어느 말이 맞는지 보여준 뒤 쓰러진 말 쪽으로 자연스럽게 이동한다.
- 운영자는 카메라 대상 주자를 직접 고를 수 있으며, 헬리콥터 연출이 발동 중일 때는 카메라 대상 변경을 잠근다.
- 첫 로딩을 줄이기 위해 헬리콥터는 외부/GLB 에셋 없이 절차형 Three.js 모델로 구성한다.

## 1차 MVP 범위

1. 참가자 입력
   - 줄 단위 참가자 이름 입력
   - 최대 500명 참가자 수 제한
   - 샘플 참가자 생성
   - 참가자별 말 색상, 무늬, 말별 구간 속도 배율 자동 부여
   - 경기별 숨김 스킬 자동 부여

2. 토너먼트 룰
   - 조당 인원 설정
   - 한 레이스 최대 20마리 제한
   - 500명까지 자동 그룹 편성
   - 조별 진출 인원 설정
   - 최종 우승 인원 설정
   - 동일 시드 기반 결과 재현

## 토너먼트 그룹 방식

- 전체 참가자 수는 1명부터 500명까지 허용한다.
- 한 레이스의 출전 수는 최대 20마리로 고정한다.
- 참가자 수가 20명 이하면 단일 결승 레이스로 진행한다.
- 참가자 수가 21명 이상이면 1라운드부터 여러 조로 나눈다.
- 조 개수는 `ceil(참가자 수 / 20)`로 계산한다.
- 각 조 인원은 가능한 균등하게 배분한다. 예를 들어 500명은 25개 조로 나뉘며, 각 조는 20명 수준으로 편성된다.
- 각 조의 상위 N마리가 다음 라운드에 진출한다.
- 다음 라운드 진출자가 20명을 넘으면 같은 방식으로 다시 조를 나눈다.
- 다음 라운드 진출자가 20명 이하가 되면 최종 결승 레이스로 진행한다.
- 최종 결승 상위 N명이 최종 우승자가 된다.
- 기본값은 조별 상위 2마리 진출, 최종 우승 1명으로 둔다.
- 운영자는 조별 진출 수와 최종 우승자 수를 변경할 수 있다.
- 라운드마다 시드를 고정해 같은 참가자 명단과 설정이면 같은 토너먼트 편성과 결과가 재현된다.

3. 3D 레이스 화면
   - Three.js 기반 3D 트랙
   - 참가자별 3D 말 모델
   - 말 위에 탑승한 단순 기수 모델
   - 레이스 진행에 따른 위치 이동
   - 출발선, 결승선, 관중석, 트랙 장식
   - 실시간 순위 표시
   - 경기 전에는 스킬명 숨김
   - 스킬 발동 순간 스킬명과 발동 상태 표시
   - 스킬 발동 시 기수 포즈와 말 속도 상승 연출

4. 카메라 연출
- 기본 카메라는 출발 지점부터 살아 있는 선두 말을 따라간다.
- 사용자가 특정 주자를 선택하면 일반 주행 중에는 해당 주자를 따라간다.
- 헬리콥터 등장/탈락 시점에는 선두 추적에서 헬리콥터와 타깃 기수 중심의 컷으로 전환한다.
   - 결승선 근접 카메라
   - 우승자 발표 카메라

5. 결과 화면
   - 토너먼트 라운드 요약
   - 결승 순위
   - 최종 우승자 목록
   - CSV 내보내기

## 3D 모델링 방향

초기 모델은 직접 제작 가능한 절차적 저폴리 스타일로 시작한다.

- 말: 큰 눈과 과장된 비율을 가진 캐주얼 마스코트형 저폴리 모델
- 형태: 단순한 몸통, 목, 머리, 꼬리, 네 다리로 구성하되 실루엣을 멀리서도 구분하기 쉽게 만든다.
- 무늬: 단색, 얼룩, 줄무늬, 점박이, 갈기/꼬리 색상 차이
- 기수: 말 위에 올라탄 단순 인형형 모델
- 말 애니메이션: 네 다리 왕복, 몸 상하 움직임, 갈기/꼬리 흔들림, 속도별 보폭 변화
- 기수 기본 애니메이션: 말의 움직임에 맞춰 몸이 흔들리고 고삐를 잡는 동작
- 기수 스킬 애니메이션: 물구나무, 눕기, 춤추기, 양팔 벌리기 같은 짧고 우스꽝스러운 포즈
- 개성 부여: 참가자별 말 색상, 무늬, 등번호, 이름표
- 확장 계획: GLB 모델 교체 가능 구조

권장 구조:

- 필요할 때만 GLB/GLTF 모델을 추가하고, 기본 런타임은 에셋 다운로드 없이 실행
- `src/scene/`에 Three.js 장면 구성
- `src/game/`에 레이스/토너먼트 엔진
- `src/ui/`에 설정/결과 패널

## 기술 스택 후보

- Vite
- TypeScript
- Three.js
- React 또는 순수 Three.js
- Vitest

초기 구현은 빠른 검증을 위해 `Vite + TypeScript + Three.js`를 우선 검토한다. React는 UI 복잡도가 커지는 시점에 도입 여부를 다시 판단한다.

## 레이스 룰 초안

- 레이스는 고정 시간 시뮬레이션으로 계산한다.
- 화면 재생 속도는 연출 속도만 바꾸고 결과 자체는 시드로 고정한다.
- 모든 말은 말별 구간 속도 배율, 경기별 숨김 배정 스킬, 현재 속도 구간, 현재 레인, 스킬 상태를 가진다.
- 목표 속도는 참가자 능력치가 아니라 말별 구간 속도 배율로 결정한다.
- 모든 말은 같은 기본 완주 시간에서 시작하고, 구간 배율의 조합으로 완주 시간이 달라진다.
- 스킬은 결과 보정이 아니라 경기 중 시각 연출로만 사용한다.
- 조별 레이스 상위 N명이 다음 라운드에 진출한다.
- 라운드 진출자가 20명을 초과하면 다음 조별 라운드를 자동 생성한다.
- 결승 레이스 상위 N명이 최종 우승자가 된다.

## 화면 구성 초안

- 상단: 프로젝트명, 현재 라운드, 시간, 주요 조작 버튼
- 중앙: 풀스크린에 가까운 3D 레이스 장면
- 하단: 실시간 순위와 참가자 상태
- 설정 레이어: 참가자/토너먼트/준비 버튼/스킬/연출 옵션
- 참가자 카드: 이름, 말 색상, 무늬
- 레이스 중계: 스킬 발동 시점에만 스킬명과 효과를 공개
- 결과 레이어: 토너먼트 진행표, 결승 순위, 최종 우승자

## 검증 기준

- 브라우저에서 3D 장면이 빈 화면 없이 렌더링된다.
- 참가자가 실제로 트랙 위를 이동한다.
- 사람 캐릭터가 직접 달리지 않고, 말이 주자로 표현된다.
- 말마다 색상과 무늬가 구분된다.
- 최대 500명 참가자 입력이 가능하다.
- 한 레이스에 최대 20마리 제한이 적용된다.
- 21명 이상 참가자는 조별 토너먼트로 자동 편성된다.
- 500명 입력 시 1라운드가 25개 조로 생성된다.
- 거리와 마장은 표시/시드 조건으로 유지하되 참가자별 능력치 유불리를 만들지 않는다.
- 참가자마다 20개 논리 주행 구간의 랜덤 속도 배율이 배정된다.
- 같은 시드와 같은 참가자 명단이면 참가자별 구간 속도 배율이 재현된다.
- 말별 구간 속도 배율에 따라 초반/중반/종반 위치 변화가 다르게 보이고 역전이 발생할 수 있다.
- 경기마다 출전 말마다 숨김 랜덤 스킬 1개가 배정된다.
- 경기 전 UI에서는 참가자의 스킬을 공개하지 않는다.
- 발동하지 않은 스킬은 경기 종료 후에도 공개하지 않는다.
- 같은 시드와 같은 참가자 명단이면 경기별 스킬 배정이 재현된다.
- 같은 참가자라도 라운드나 조가 달라지면 다른 스킬을 받을 수 있다.
- 스킬은 경기당 최대 1회만 발동한다.
- 6개 MVP 스킬은 서로 다른 발동 구간, 포즈, 이펙트를 가진다.
- 스킬 발동 시 5초 동안 기수 특수 포즈가 보인다.
- 레이스 시작 시 설정/결과 패널이 자동으로 숨겨지고, 경기 중 상단 버튼으로 다시 열고 닫을 수 있다.
- 사용자가 옵션과 참가자를 고른 뒤 `시작`을 누르기 전까지 말들은 출발선에서 대기한다.
- 트랙은 초기 긴 직선 주로보다 약 3배 더 길게 보이고, 말이 달리는 실제 이동 거리와 레이스 재생 시간이 짧게 끝나지 않는다.
- 트랙 자체에는 고정 속도 구간을 두지 않고, 말마다 20개 논리 주행 구간의 시드 기반 랜덤 배속을 가져 실제 주행 속도와 다리 모션 속도가 변한다.
- 기본 카메라는 출발 지점부터 선두 말을 따라가며, 이벤트가 없을 때 고정 전체 뷰에 머무르지 않는다.
- 말은 트랙의 결승선 방향을 바라보며 달린다.
- 말은 장난감형 단순 몸통이 아니라 가슴, 엉덩이, 어깨, 관절이 강조된 근육질 실루엣으로 보인다.
- 말마다 호다닥 달리기, 일반 달리기, 걷기, 산책 수준의 다리 모션이 다르게 배정된다.
- 기수는 머리, 몸통, 팔, 다리가 구분되는 사람형 저폴리 모델로 보인다.
- 헬리콥터 탈락 이벤트가 발생하면 한 레이스에서 출전자의 약 1/3이 무작위 타깃이 되고, 해당 말들은 레이스에서 진출/우승할 수 없다.
- 헬리콥터는 시작부터 보이지 않고 선두가 1/3 지점을 지날 때 골인지점 쪽에서 등장하며, 선두가 1/2 지점을 지날 때부터 충분한 간격을 두고 탈락 연출을 순차 재생한다.
- 첫 등장 이후에는 같은 골인지점 위치에 머문 채 다음 타깃을 조준한다.
- 헬리콥터 연출 중에는 카메라 대상 변경 UI가 잠긴다.
- 헬리콥터 이벤트는 GLB 에셋으로 렌더링되고, 총구 섬광, 총알 궤적, 총알 추적 카메라, 순간 슬로우모션, 피격 상태, 탈락 상태가 화면에 표시된다.
- 헬리콥터 탈락 이후 해당 말은 트랙 위에 쓰러진 상태로 남는다.
- 최소 데스크톱/모바일 뷰포트에서 UI가 겹치지 않는다.
- 같은 시드와 같은 설정이면 같은 최종 우승자가 나온다.
- 토너먼트 설정이 잘못되면 명확한 오류를 낸다.
- 최종 우승자 수가 설정값과 일치한다.
