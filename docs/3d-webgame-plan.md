# 3D 웹게임 기획서

## 목표

`run-hoban-run`은 브라우저에서 실행되는 3D 레이스 연출형 웹게임으로 새로 제작한다.

기존 경마 추첨 게임 구현은 폐기하고, 다양한 색상과 무늬의 말이 달리는 3D 레이스, 관전 연출, 토너먼트 진행, 최종 우승자 발표를 중심으로 다시 설계한다.

## 레퍼런스 방향

`우마무스메`는 다음 관점에서만 참고한다. 단, 이 프로젝트의 주자는 사람이 아니라 말이다.

- 3D 레이스의 속도감과 카메라 박력
- 경기 전후 연출과 카메라 컷 전환
- 참가자별 개성과 능력치가 결과에 영향을 주는 구조
- 자동 레이스를 관전하며 순위 변화와 스킬 발동을 따라가는 구조
- 관전자가 현재 순위와 역전 흐름을 쉽게 이해하는 UI
- 최종 우승자 발표의 무대감

주의할 점:

- 우마무스메의 캐릭터, 세계관, 명칭, 의상, UI, 음원, 모델, 애니메이션을 복제하지 않는다.
- 사람 캐릭터가 직접 달리는 구조를 사용하지 않는다.
- 자체 캐릭터 콘셉트와 자체 룰을 사용한다.
- 참고는 제품 방향성과 연출 품질 기준으로만 사용한다.

## 핵심 게임 콘셉트

- 장르: 3D 레이스 시뮬레이션 + 행사 추첨/토너먼트 게임
- 실행 환경: 데스크톱 브라우저 우선, 대형 현장 디스플레이 대응
- 플레이 방식: 운영자가 참가자 명단과 토너먼트 옵션을 입력하면 3D 레이스가 자동 진행
- 결과 방식: 조별 레이스를 거쳐 결승 참가자를 선발하고, 결승 상위 N명을 최종 우승자로 선정
- 주자 콘셉트: 참가자별로 색상, 무늬, 번호가 다른 말
- 스킬 콘셉트: 말 위의 기수가 스킬 발동 순간 과장된 포즈와 짧은 퍼포먼스를 보여주며 속도 상승 효과를 만든다.
- 비주얼 스타일: 현실 비율보다 귀엽고 알아보기 쉬운 캐주얼 마스코트형 말
- 레이스 규모: 한 레이스는 최대 18마리까지 출전한다.
- 참가자 규모: 전체 참가자는 최대 500명까지 입력할 수 있고, 초과 인원은 받지 않는다.
- 진행 방식: 18마리를 넘는 참가자는 조별 토너먼트 라운드로 자동 편성한다.

## 우마무스메식 기본 룰 변환

원본 게임의 수치 공식, 고유 스킬, 캐릭터 시스템은 복제하지 않는다. 대신 기본 레이스 구조를 단순화해 자체 말 레이스 룰로 변환한다.

### 레이스 조건

- 레이스는 거리, 마장, 방향, 트랙 상태를 가진다.
- 거리 타입은 단거리, 마일, 중거리, 장거리로 나눈다.
- 마장은 잔디와 더트로 시작한다.
- 방향은 좌회전, 우회전으로 둔다.
- 트랙 상태는 양호, 다습, 불량 정도로 단순화한다.
- 거리와 트랙 상태는 소모 체력, 목표 속도, 가속에 영향을 준다.

### 말 능력치

- 참가자마다 속도, 체력, 파워, 근성, 지능 능력치를 랜덤으로 부여한다.
- 능력치 배정은 시드 기반으로 계산해 같은 참가자 명단과 같은 시드에서는 항상 재현된다.
- 완전한 압도 차이가 나지 않도록 모든 말은 같은 총 능력치 예산 안에서 분배한다.
- 속도: 최고 속도와 라스트 스퍼트 속도에 가장 크게 영향을 준다.
- 체력: 레이스 전체의 에너지이며, 부족하면 라스트 스퍼트가 늦어지거나 약해진다.
- 파워: 출발 가속, 코너 이후 재가속, 추월 시 위치 변경 성공률에 영향을 준다.
- 근성: 결승 직선에서 비슷한 속도의 말끼리 붙었을 때 버티는 힘과 막판 유지력에 영향을 준다.
- 지능: 스킬 발동 타이밍, 레이스 중 위치 안정성, 불리한 움직임 발생률에 영향을 준다.

### 적성

- 말마다 마장 적성, 거리 적성, 작전 적성을 가진다.
- 적성은 `S`, `A`, `B`, `C`, `D` 등급으로 표현한다.
- `A`를 기준값으로 보고, `S`는 소폭 보너스, `B` 이하는 페널티를 준다.
- 마장 적성은 가속과 코너 안정성에 영향을 준다.
- 거리 적성은 목표 속도와 라스트 스퍼트 효율에 영향을 준다.
- 작전 적성은 해당 작전을 선택했을 때 위치 안정성과 판단 품질에 영향을 준다.

### 작전

- 선두형: 초반부터 1위권을 잡고 끝까지 버티는 작전이다. 속도와 체력이 중요하다.
- 선행형: 선두 바로 뒤에서 안정적으로 따라가다가 중후반에 압박하는 작전이다.
- 추격형: 중단에서 체력을 아끼며 달리다가 종반에 가속하는 작전이다.
- 막판형: 후방에서 기다리다가 마지막 구간에서 크게 치고 올라오는 작전이다. 체력, 파워, 근성이 중요하다.

### 레이스 페이즈

- 출발: 게이트 지연을 작은 랜덤값으로 계산하고, 짧은 출발 가속을 적용한다.
- 초반: 작전에 맞는 위치를 잡는다.
- 중반: 체력 소모를 관리하며 위치를 유지하거나 추월을 시도한다.
- 종반: 남은 체력을 확인하고 스퍼트 진입 시점을 결정한다.
- 라스트 스퍼트: 체력이 충분한 말은 목표 속도를 크게 올리고, 부족한 말은 스퍼트가 늦거나 약해진다.
- 결승 직선 접전: 가까운 거리에서 비슷한 속도로 달리는 말끼리는 근성 기반 접전 보정을 받는다.

### 스킬

- 장애물 횟수와 통과 확률 시스템은 사용하지 않는다.
- 스킬 기본 발생 확률은 3%로 고정한다.
- 스킬이 발동하면 5초 동안 말의 속도가 상승한다.
- 스킬은 경기당 최대 1회만 발동한다.
- 스킬별로 속도 상승폭, 발동 구간, 기수 포즈, 이펙트, 중계 문구에 차이를 둔다.
- 스킬 발동 중에는 기수가 물구나무, 눕기, 춤추기, 양팔 벌리기 같은 특수 포즈를 취한다.
- 지능은 MVP에서 확률 자체를 바꾸지 않고, 추후 스킬이 더 유리한 타이밍에 발동하는 보정값으로만 확장한다.

### 경기별 숨김 랜덤 스킬 배정

- 참가자 입력이 확정되면 각 참가자에게 말 프로필을 자동 부여한다.
- 스킬은 참가자에게 영구 고정하지 않고, 매 경기 시작 시 출전 말마다 1개씩 숨김 배정한다.
- 참가자는 경기 전 자신이 어떤 스킬을 받았는지 알 수 없다.
- 스킬은 레이스 중 실제로 발동되는 순간 연출과 중계 문구로 처음 공개된다.
- 발동하지 않은 스킬은 경기 종료 후에도 공개하지 않는다.
- 다음 라운드 또는 다음 경기에서는 같은 참가자라도 다른 스킬을 받을 수 있다.
- 스킬 배정은 랜덤이지만 시드 기반으로 계산해 같은 참가자 명단, 같은 토너먼트 설정, 같은 시드에서는 경기별 스킬 배정이 재현된다.
- 경기별 스킬 시드는 `전체 시드 + 라운드 번호 + 조 번호 + 참가자 ID`를 조합해 만든다.
- MVP는 스킬 6개로 시작한다.
- 모든 스킬은 공통으로 `3% 발동 확률`, `5초 지속`, `경기당 최대 1회 발동` 규칙을 따른다.
- 스킬별 차이는 속도 상승폭, 발동 가능 구간, 기수 포즈, 이펙트 색상, 중계 문구로 표현한다.
- 추후 확장 시 스킬 개수, 발동 조건, 희귀도를 늘릴 수 있다.

MVP 스킬 풀:

- 로켓 스타트: 출발/초반 구간에서 발동한다. 속도 상승폭은 높지만 초반에만 가능하다. 기수가 물구나무 포즈를 취하고 짧은 불꽃 이펙트를 표시한다.
- 코너 댄스: 코너 또는 중반 구간에서 발동한다. 속도 상승폭은 중간이며 코너 감속을 줄인다. 기수가 춤추는 포즈를 취하고 리듬감 있는 별 이펙트를 표시한다.
- 드러눕기 질주: 중반 이후 발동한다. 속도 상승폭은 중간보다 높고 체력 소모가 조금 늘어난다. 기수가 말 위에 누운 포즈를 취하고 바람줄기 이펙트를 표시한다.
- 결승선 광란: 종반/라스트 스퍼트 구간에서 발동한다. 속도 상승폭은 가장 높지만 발동 구간이 짧다. 기수가 양팔을 벌리고 환호하는 포즈를 취한다.
- 진흙 튀기기: 더트 또는 불량 트랙에서 발동한다. 속도 상승폭은 중간이며 더트/불량 조건에서 보너스가 커진다. 말 뒤쪽에 흙먼지 이펙트를 표시한다.
- 잔디 미끄럼: 잔디 또는 양호 트랙에서 발동한다. 속도 상승폭은 중간이며 잔디/양호 조건에서 보너스가 커진다. 말 발밑에 녹색 잔상 이펙트를 표시한다.

## 1차 MVP 범위

1. 참가자 입력
   - 줄 단위 참가자 이름 입력
   - 최대 500명 참가자 수 제한
   - 샘플 참가자 생성
   - 참가자별 말 색상, 무늬, 작전, 랜덤 능력치 자동 부여
   - 경기별 숨김 스킬 자동 부여

2. 토너먼트 룰
   - 조당 인원 설정
   - 한 레이스 최대 18마리 제한
   - 500명까지 자동 그룹 편성
   - 조별 진출 인원 설정
   - 최종 우승 인원 설정
   - 동일 시드 기반 결과 재현

## 토너먼트 그룹 방식

- 전체 참가자 수는 1명부터 500명까지 허용한다.
- 한 레이스의 출전 수는 최대 18마리로 고정한다.
- 참가자 수가 18명 이하면 단일 결승 레이스로 진행한다.
- 참가자 수가 19명 이상이면 1라운드부터 여러 조로 나눈다.
- 조 개수는 `ceil(참가자 수 / 18)`로 계산한다.
- 각 조 인원은 가능한 균등하게 배분한다. 예를 들어 500명은 28개 조로 나뉘며, 각 조는 17명 또는 18명 수준으로 편성된다.
- 각 조의 상위 N마리가 다음 라운드에 진출한다.
- 다음 라운드 진출자가 18명을 넘으면 같은 방식으로 다시 조를 나눈다.
- 다음 라운드 진출자가 18명 이하가 되면 최종 결승 레이스로 진행한다.
- 최종 결승 상위 N명이 최종 우승자가 된다.
- 기본값은 조별 상위 2마리 진출, 최종 우승 1명으로 둔다.
- 운영자는 조별 진출 수와 최종 우승자 수를 변경할 수 있다.
- 라운드마다 시드를 고정해 같은 참가자 명단과 설정이면 같은 토너먼트 편성과 결과가 재현된다.

3. 3D 레이스 화면
   - Three.js 기반 3D 트랙
   - 참가자별 3D 말 모델
   - 말 위에 탑승한 단순 기수 모델
   - 레이스 진행에 따른 위치 이동
   - 출발선, 결승선, 관중석, 트랙 장식
   - 실시간 순위 표시
   - 경기 전에는 스킬명 숨김
   - 스킬 발동 순간 스킬명과 발동 상태 표시
   - 스킬 발동 시 기수 포즈와 말 속도 상승 연출

4. 카메라 연출
   - 전체 트랙을 보는 기본 카메라
   - 선두권 추적 카메라
   - 결승선 근접 카메라
   - 우승자 발표 카메라

5. 결과 화면
   - 토너먼트 라운드 요약
   - 결승 순위
   - 최종 우승자 목록
   - CSV 내보내기

## 3D 모델링 방향

초기 모델은 직접 제작 가능한 절차적 저폴리 스타일로 시작한다.

- 말: 큰 눈과 과장된 비율을 가진 캐주얼 마스코트형 저폴리 모델
- 형태: 단순한 몸통, 목, 머리, 꼬리, 네 다리로 구성하되 실루엣을 멀리서도 구분하기 쉽게 만든다.
- 무늬: 단색, 얼룩, 줄무늬, 점박이, 갈기/꼬리 색상 차이
- 기수: 말 위에 올라탄 단순 인형형 모델
- 말 애니메이션: 네 다리 왕복, 몸 상하 움직임, 갈기/꼬리 흔들림, 속도별 보폭 변화
- 기수 기본 애니메이션: 말의 움직임에 맞춰 몸이 흔들리고 고삐를 잡는 동작
- 기수 스킬 애니메이션: 물구나무, 눕기, 춤추기, 양팔 벌리기 같은 짧고 우스꽝스러운 포즈
- 개성 부여: 참가자별 말 색상, 무늬, 등번호, 이름표
- 확장 계획: GLB 모델 교체 가능 구조

권장 구조:

- `public/models/`에 GLB/GLTF 모델 배치
- `src/scene/`에 Three.js 장면 구성
- `src/game/`에 레이스/토너먼트 엔진
- `src/ui/`에 설정/결과 패널

초기 구현은 빠른 검증을 위해 `Vite + TypeScript + Three.js`를 우선 검토한다. React는 UI 복잡도가 커지는 시점에 도입 여부를 다시 판단한다.

## 레이스 룰 초안

- 레이스는 고정 시간 시뮬레이션으로 계산한다.
- 화면 재생 속도는 연출 속도만 바꾸고 결과 자체는 시드로 고정한다.
- 모든 말은 능력치, 적성, 작전, 경기별 숨김 배정 스킬, 현재 체력, 현재 속도, 현재 레인, 스킬 상태를 가진다.
- 목표 속도는 레이스 페이즈, 작전, 거리 적성, 현재 체력으로 결정한다.
- 현재 속도는 목표 속도까지 서서히 변하고, 파워가 높을수록 더 빨리 가속한다.
- 체력은 거리, 현재 속도, 트랙 상태에 따라 줄어든다.
- 라스트 스퍼트는 남은 체력이 충분할 때 강하게 발동하고, 부족하면 늦게 발동하거나 속도 상승폭이 줄어든다.
- 추월은 앞말과 거리, 레인 여유, 파워, 지능을 기준으로 판정한다.
- 조별 레이스 상위 N명이 다음 라운드에 진출한다.
- 라운드 진출자가 18명을 초과하면 다음 조별 라운드를 자동 생성한다.
- 결승 레이스 상위 N명이 최종 우승자가 된다.

## 화면 구성 초안

- 상단: 프로젝트명, 현재 라운드, 시간, 주요 조작 버튼
- 중앙: 풀스크린에 가까운 3D 레이스 장면
- 하단: 실시간 순위와 참가자 상태
- 설정 레이어: 참가자/토너먼트/시드/스킬/연출 옵션
- 참가자 카드: 이름, 말 색상, 무늬, 작전
- 레이스 중계: 스킬 발동 시점에만 스킬명과 효과를 공개
- 결과 레이어: 토너먼트 진행표, 결승 순위, 최종 우승자

## 검증 기준

- 브라우저에서 3D 장면이 빈 화면 없이 렌더링된다.
- 참가자가 실제로 트랙 위를 이동한다.
- 사람 캐릭터가 직접 달리지 않고, 말이 주자로 표현된다.
- 말마다 색상과 무늬가 구분된다.
- 최대 500명 참가자 입력이 가능하다.
- 한 레이스에 최대 18마리 제한이 적용된다.
- 19명 이상 참가자는 조별 토너먼트로 자동 편성된다.
- 500명 입력 시 1라운드가 28개 조로 생성된다.
- 거리, 마장, 작전, 능력치에 따라 레이스 흐름이 달라진다.
- 참가자마다 랜덤 능력치가 배정된다.
- 같은 시드와 같은 참가자 명단이면 참가자별 능력치 배정이 재현된다.
- 체력이 부족한 말은 라스트 스퍼트가 늦거나 약해진다.
- 작전에 따라 초반/중반/종반 위치 변화가 다르게 보인다.
- 경기마다 출전 말마다 숨김 랜덤 스킬 1개가 배정된다.
- 경기 전 UI에서는 참가자의 스킬을 공개하지 않는다.
- 발동하지 않은 스킬은 경기 종료 후에도 공개하지 않는다.
- 같은 시드와 같은 참가자 명단이면 경기별 스킬 배정이 재현된다.
- 같은 참가자라도 라운드나 조가 달라지면 다른 스킬을 받을 수 있다.
- 스킬은 경기당 최대 1회만 발동한다.
- 6개 MVP 스킬은 서로 다른 발동 구간, 속도 상승폭, 포즈, 이펙트를 가진다.
- 스킬 발동 시 5초 속도 상승과 기수 특수 포즈가 함께 보인다.
- 최소 데스크톱/모바일 뷰포트에서 UI가 겹치지 않는다.
- 같은 시드와 같은 설정이면 같은 최종 우승자가 나온다.
- 토너먼트 설정이 잘못되면 명확한 오류를 낸다.
- 최종 우승자 수가 설정값과 일치한다.
