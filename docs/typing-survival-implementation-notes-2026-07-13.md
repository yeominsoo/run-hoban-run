# 타이핑 생존 구현 노트 (2026-07-13)

## 원본 스펙 (v1)

- 화면 상단에서 떨어지는 단어를 바닥에 닿기 전에 타이핑
- 단어 소스: 내장 단어 목록(영어 100개 + 한국어 100개)
- 레벨: 10초마다 속도 업, 동시 단어 수 증가
- 종료 조건: 단어가 바닥에 닿으면 HP -1 (HP 3)
- 점수: 단어 제거 수 × 레벨 배수
- Canvas(단어 애니메이션) + DOM input(입력 필드)
- 모바일: `visualViewport` 기준 높이 재계산, 모바일은 영어 단어 전용 모드 권장

## 결정과 이유

1. **매칭 방식**: 입력 필드에 키를 입력할 때마다(각 `input` 이벤트마다) 현재 입력값과
   화면에 떠 있는 모든 단어를 대소문자 무시하고 정확히 비교한다. 일치하는 단어가 있으면
   즉시 제거하고 입력창을 비운다. Enter나 별도 제출 버튼을 요구하지 않는다 — 타이핑
   생존류 게임(ZType 등)의 표준 UX가 실시간 매칭이고, 제출 버튼을 추가하면 오히려
   "생존"의 긴박감이 떨어진다.
2. **동일 단어 중복 시 우선순위**: 화면에 같은 텍스트의 단어가 두 개 이상 떠 있으면
   바닥에 더 가까운(=더 급한) 단어를 제거한다.
3. **속도/동시 단어 수 증가 곡선**: 10초마다 낙하 속도 ×1.12, 최대 동시 단어 수 +1(최대
   6개까지). 초기값은 속도 40px/s, 최대 동시 단어 1개로 시작 — 스펙에 구체 수치가 없어서
   볼 피하기의 "30초마다 ×1.2" 패턴보다 살짝 완만한 배율을 골랐다. 이 게임은 조건 충족
   시(레벨 상승) 동시에 화면의 단어 개수까지 늘어나 체감 난이도가 속도 하나만 오르는
   게임보다 훨씬 가파르게 오르기 때문에, 배율 자체는 더 보수적으로 잡았다.
4. **단어 언어 비율**: 매 스폰마다 영어/한국어 풀에서 50:50으로 무작위 선택한다(모바일
   제외, 아래 6번 참고). 한쪽으로 치우치면 "영어 100개 + 한국어 100개"를 준비한 의미가
   없어진다.
5. **점수 공식**: "단어 제거 수 × 레벨 배수"를 그대로 따라, 단어를 제거할 때마다
   `현재 레벨`만큼 점수를 더한다(레벨 = 10초 경과 단계 수 + 1). 레벨이 오를수록 단어 하나당
   가치가 커져, 뒷심을 발휘할수록 보상이 커지는 구조.
6. **모바일 자동 영어 전용 모드**: `window.matchMedia('(pointer: coarse)')`로 터치 주입력
   기기를 감지해, 참이면 한국어 단어를 스폰 풀에서 제외한다. 스펙이 권장한 그대로이며,
   이유도 스펙에 있는 그대로다 — 모바일 소프트 키보드에서 한글 입력(조합 중 상태 처리)은
   영어 입력보다 오탐지 소지가 커서 생존 게임의 빠른 판정과 궁합이 나쁘다.
7. **`visualViewport` 대응**: 소프트 키보드가 열리면 `window.visualViewport.height`가
   줄어드는데, 이 값을 캔버스 높이 계산에 반영해 키보드에 가려지지 않는 영역에서만 단어가
   떨어지도록 한다. `visualViewport`를 지원하지 않는 환경(구형 브라우저)에서는
   `window.innerHeight`로 자연스럽게 폴백한다.
8. **입력 포커스 유지**: 게임 중 입력 필드가 blur되면(예: 실수로 다른 곳 탭) 즉시 다시
   포커스를 맞춘다 — 안 그러면 키 입력이 아예 씹혀서 억울하게 단어를 놓치게 된다.
