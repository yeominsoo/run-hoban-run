# 안엘런 구현 노트 (최종 갱신 2026-07-21)

## 원본 스펙 (v1)

- 목표: 자동으로 달리는 캐릭터가 장애물을 피하며 최대한 오래 달리기
- 초기 입력안: 탭/클릭 = 점프, 스와이프 다운 / 빠른 탭 2회 = 슬라이드
- 규칙: 속도 지속 상승, 코인 수집 시 +10점
- 장애물: 낮은 장애물(점프), 높은 장애물(슬라이드), 구덩이(타이밍 점프)
- 점수: 달린 거리(1m = 1점) + 코인
- Canvas 사용, 모바일 주의사항 없음

## 현재 플레이 규칙 (2026-07-21)

- 탭/클릭/위 키: 1단 점프, 공중에서 한 번 더 입력하면 2단 점프
- 아래 스와이프: 시간제 슬라이드, 아래 키: 누르는 동안 유지하고 놓으면 즉시 일어서기
- 코인은 독립적인 임의 높이가 아니라 주변 장애물의 안전 회피 동선에 맞춰 정렬
  - 낮은 장애물·구덩이 주변: 점프 궤도
  - 높은 장애물 주변: 슬라이드 궤도
  - 이후 장애물이 생성돼도 135px 안의 기존 코인을 해당 회피 궤도로 재정렬
- 15초마다 라운드 상승. 속도·장애물 밀도·구덩이 출현 비율이 6단계까지 점진적으로 증가
- 낮은 장애물은 보강 철물 상자, 높은 장애물은 케이블식 경고 장벽, 구덩이는 암벽 깊이로 표현

## 결정과 이유

1. **목숨 수: 1개(단일 충돌 즉시 종료)**. 스펙에 HP/목숨 언급이 없다. 무한 러너 장르
   표준(Chrome Dino 등)이 "한 번 부딪히면 끝"이라 별도 HP 시스템을 새로 만들지 않았다 —
   다른 게임들(HP 3)과 다른 이유가 명확할 때만 다르게 가는 편이 낫다고 판단했다.
2. **15초 라운드와 라운드 내부 연속 가속을 결합**: 라운드가 바뀌면 기준 속도와 장애물
   밀도가 한 단계 올라가고, 같은 라운드 안에서는 속도가 부드럽게 상승한다. 난이도 변화
   시점을 HUD에서 예측할 수 있으면서도 주행 감각이 갑자기 끊기지 않게 했다.
3. **점프는 물리 기반(중력+초기속도), 슬라이드는 판정 박스 축소+타이머**: 점프 궤적은
   `y' = v, v' = -GRAVITY`로 시뮬레이션해 자연스러운 포물선을 만들고, 슬라이드는 별도
   물리 없이 "일정 시간 동안 캐릭터 높이만 낮아짐" 상태로 단순화했다 — 슬라이드까지
   물리로 구현하면 복잡도만 늘고 체감 차이가 없다.
4. **두 번째 탭은 2단 점프**: 첫 탭은 1단 점프, 공중의 두 번째 탭은 수직 속도를 다시 주는
   2단 점프로 사용한다. 세 번째 입력은 무시한다. 슬라이드는 아래 스와이프로 명확히 분리해
   탭 조합이 점프와 슬라이드 중 어느 쪽인지 뒤늦게 바뀌지 않게 했다.
5. **장애물 3종의 구체적 모양**:
   - 낮은 장애물: 바닥에 붙은 블록 → 점프해야 통과
   - 높은 장애물: 머리 높이에 떠 있는 막대 → 슬라이드해야 통과(서 있으면 부딪힘)
   - 구덩이: 바닥 자체가 끊긴 구간 → 점프 중(공중)이어야 안전, 착지 지점이 구덩이 안이면
     추락 판정
6. **구덩이 폭을 시간에 따라 넓히지 않음**: 점프의 체공 시간은 고정(물리 상수)이지만 이동
   속도는 계속 오르므로, "속도×체공시간"으로 계산되는 실제 점프 사거리는 시간이 지날수록
   저절로 늘어난다. 따라서 구덩이 폭을 고정해두면 난이도가 저절로 유지되고, 오히려 속도
   자체가 반응 시간을 줄이는 쪽으로 난이도를 올린다 — 별도로 구덩이를 더 넓힐 필요가 없다고
   판단했다.
7. **점수 환산**: "1m = 1점"을 위해 화면 픽셀과 "미터"를 잇는 임의 환산값이 필요했다 —
   `50px = 1m`로 정하고 `달린 거리 점수 = floor(누적 스크롤 거리 / 50)`으로 계산한다.
   코인은 스펙대로 개당 +10점을 별도로 더한다(거리 점수와 코인 점수를 합산해 최종 점수).
8. **라운드별 장애물 스폰**: 1라운드는 구덩이를 제외해 조작 학습 시간을 주고, 이후
   라운드마다 최소 간격을 줄이면서 구덩이 비율을 최대 30%까지 올린다. 난이도는 6단계에서
   상한을 두어 반응 불가능한 조합으로 치닫지 않게 했다.
9. **랭킹 UI**: 완료된 7개 게임과 동일한 공용 패턴(`shared/leaderboard.ts`,
   `shared/game-shell.css`)을 처음부터 포함해서 만든다.

## 자동 플레이 테스트로 발견한 버그: 점프로 높은 장애물을 넘어갈 수 있었음

"낮은 장애물은 점프, 높은 장애물은 슬라이드"라는 스펙을 구현하면서, 높은 장애물을 단순히
"바닥에서 좀 띄운 30px짜리 블록"으로만 만들었다. 그런데 점프 최고 높이가 약 131px로 이
블록(바닥에서 30~60px 높이)보다 훨씬 높아서, 타이밍만 맞으면 슬라이드 없이 그냥 점프로도
넘어가버렸다 — "장애물마다 정해진 유일한 통과 방법이 있다"는 스펙의 의도가 깨지는 버그였다.
직접 여러 장애물 유형을 골고루 만나며 15초간 살아남는 자동 플레이 Playwright 테스트를 짜서
검증하는 과정에서 발견했다(수동으로 몇 번 눌러보는 것만으로는 우연히 안 걸렸을 수도 있다).

**수정**: 높은 장애물의 충돌 판정 사각형을 "슬라이드 통로(clearance)만 남기고 화면 맨 위까지
뻗은 기둥"으로 바꿨다(`top: 0, height: groundY - clearance`). 이러면 점프의 최고 높이가
아무리 높아도 항상 기둥과 겹치기 때문에 슬라이드만이 유일한 통과 방법이 된다. 시각적으로는
위쪽을 그라디언트로 흐리게 처리해 "천장 너머로 이어지는 기둥"처럼 보이게 했다.
**교훈**: 이런 종류의 "장애물 A는 반드시 액션 X로만 피해야 한다"는 규칙은, 회피 액션이
만들 수 있는 모든 상태(이 경우 점프의 전체 높이 범위)를 실제로 계산해서 장애물의 판정
범위가 그 전체 범위를 덮는지 확인해야 한다 — 대충 "장애물을 좀 낮게 두면 되겠지"로는
부족했다.

## 코인 유도 동선과 회피 동선 충돌 개선 (2026-07-21)

코인과 장애물을 각자 다른 주기로 생성하던 구조에서는 코인이 점프를 유도한 직후 높은
장애물이 나오거나, 낮은 코인이 구덩이 앞에 놓이는 식으로 보상 동선과 생존 동선이 충돌할
수 있었다. 코인을 획득한 직후 사실상 피할 수 없는 상황이 생기는 것은 난이도가 아니라
절차 생성의 불공정성이므로 다음처럼 수정했다.

- 코인마다 `run`·`jump`·`slide` 안전 액션을 기록한다.
- 코인 생성 시 135px 안의 가장 가까운 장애물을 기준으로 높이를 결정한다.
- 장애물이 코인보다 나중에 생성돼도 주변 기존 코인을 장애물의 안전 액션에 맞춰 재정렬한다.
- 탭 2회는 점프 궤도를 한 번 더 보정할 수 있는 2단 점프로 사용하고 세 번째 점프는 막는다.
- 아래 스와이프는 공중에서도 점프를 취소하고 즉시 슬라이드로 전환할 수 있게 유지한다.

Playwright 자동 플레이는 매 프레임 코인과 135px 안의 장애물을 비교해 높은 장애물 주변
코인이 슬라이드 높이, 낮은 장애물·구덩이 주변 코인이 점프 높이에 있는지 검증한다. 같은
테스트에서 2라운드까지 실제로 생존하는지 확인하며, 별도 입력 테스트는 두 번째 탭이 2단
점프가 되고 세 번째 탭이 무시되는지 고정한다.

## 신규 캐릭터 8프레임 액션 시트 채택 (2026-07-14)

최신 `v3-two-girls-dress-corrected` 디자인 3장을 기준으로 꽃모자 소녀와 분홍안경 소녀를
세 스타일씩 분리해 총 6종을 실제 플레이어 이미지로 채택했다. 이전 v2 기반 구현에서
꽃모자 캐릭터를 소년으로 바꾸고 분홍안경 캐릭터의 A라인 원피스를 바지처럼 만든 오류도
함께 수정했다. 선택값은 `rhh_endless-runner_character`에 저장한다.

원본 시트는 생성 당시의 임시 이름 대신 다음 이름으로 보존한다.

- `runner-flat-sticker-action-sheet.png`
- `runner-storybook-paper-action-sheet.png`
- `runner-soft-3d-toy-action-sheet.png`

### 실제 8프레임 원화와 GIF 제작 원칙

- 최신 v3 시트는 캐릭터의 얼굴·복장·비율·스타일을 고정하는 디자인 참조로 사용했다.
- 내장 이미지 생성 기능으로 캐릭터마다 `run-jump`, `slide-fall` 4×4 시트를 한 장씩 만들었다.
  각 액션은 두 행에 걸친 시간 순서의 실제 8포즈이며, 총 시트 12장과 원화 192장이다.
- 팔·다리·몸 중심·실루엣·옷 주름이 실제로 다른 그림이다. 이동·회전·스케일·미러·복제
  보간을 사용하지 않는다.
- 생성 시트 배경은 정확한 `#00ff00`으로 정규화했다. imagegen 크로마키 도구의 border
  자동 샘플링, soft matte, despill, 1px edge contract로 투명화한다.
- 공중·저자세 포즈가 명목상 4×4 셀 경계를 넘는 경우가 있어 고정 셀 크롭을 사용하지 않는다.
  전체 시트의 연결된 완전한 피사체 16개를 검출하고, 중심점으로 4×4 슬롯에 배정한 뒤 각
  피사체 전체 경계를 크롭한다. 이 방식으로 이웃 셀 조각과 포즈 잘림을 제거했다.
- 분리한 192개 원화를 `256×256`, 발 기준 피벗 `(128, 232)`에 정규화하고 각 동작의 8장을
  순서대로 GIF에 직접 인코딩했다.
- 유사도 감사에서 분홍안경 동화책 FALL 7·8이 지나치게 비슷해 FALL 8을 새로 그린
  옆누움·굽힌 다리 포즈로 교체했다. 같은 시트의 다른 15개 피사체는 변경하지 않았고 해당 쌍의
  실루엣 IoU는 `0.9711`에서 `0.8179`로 낮아졌다.
- 달리기는 640ms 반복, 점프는 물리 체공시간에 맞춘 690ms, 넘어짐은 1,100ms다.
- 슬라이드는 `enter` 2장/180ms, `hold` 4장/400ms 반복, `exit` 2장/180ms로 분리했다.
- 캐릭터 폴더의 액션 PNG는 첫 프레임 미리보기이며 `frames/`에 01~08 PNG를 보존한다.
- `tools/build_endless_runner_character_assets.py`로 크로마키·프레임 분리·GIF·manifest 생성을
  재현할 수 있고, `tools/verify_endless_runner_character_assets.py`가 8프레임 계약과 구형
  산출물 부재를 검사한다.

### 게임 상태 연결

Canvas의 물리·충돌 판정 크기(`PLAYER_WIDTH`, `PLAYER_HEIGHT`, `SLIDE_HEIGHT`)는 그대로 두고,
시각 캐릭터만 DOM `<img>` GIF로 분리했다. 게임 상태는 `running → run`, `jumping → jump`,
`falling → fall`로 연결한다. `sliding` 안에는 `enter → hold → exit` 단계가 있으며 기본
1,160ms 동안 재생한다. 슬라이드 중 추가 입력은 낮은 `hold` 단계만 800ms 연장한다. 충돌하면
월드 이동·점수·입력을 정지한 `falling` 상태로 전환해 1.1초 동안 넘어짐 GIF를 재생한다.

## 체크 조끼 소년 3스타일 확장 (2026-07-15)

기존 두 소녀 6종은 픽셀 단위로 보존하고, 동일한 남자아이를 촬영한 사진 3장을 정체성 참조로
사용해 `checkered-vest-boy`를 세 렌더링 스타일로 추가했다. 제작·재생성용 자산 인벤토리는
세 인물 × 세 스타일의 9종으로 보존한다. 2026-07-21 품질 개선에서 최종 런타임 범위는
분홍안경 소녀와 체크 조끼 소년의 소프트 3D 토이 버전 2종으로 승격했다.

### 정체성·복장 고정

- 눈·얼굴 최우선 참조는 정면에 가까운 세 번째 사진이다. 얼굴 대비 큰 편의 짙은 무쌍 눈,
  보이는 홍채와 동공, 자연스러운 위·아래 눈 윤곽을 유지한다.
- 쌍꺼풀 선, 작은 점눈, 웃을 때 항상 감기는 반달 선 눈은 금지한다. 눈웃음이 있어도 눈동자가
  읽히도록 열어 두고, 과장된 애니메이션 눈으로 키우지는 않는다.
- 짧은 검정 보울컷과 눈썹 위 일자 앞머리, 둥근 볼, 작은 코, 밝은 치아 미소를 세 스타일과
  모든 동작 프레임에서 같은 인물의 특징으로 유지한다.
- 대표 복장은 흰 반소매 칼라 셔츠, 별도 V넥 회색 체크 조끼, 같은 체크의 무릎 위 반바지,
  검은 발목 양말, 무브랜드 검정 러닝화와 미색 밑창이다. 체크는 최종 256px에서도 읽히도록
  조끼 폭 기준 4~5칸의 중간 크기 윈도페인 패턴으로 단순화한다.
- 다른 사진의 검정 재킷, 숫자 풍선, 스튜디오 배경과 소품은 캐릭터 디자인에 포함하지 않는다.

### 디자인과 런타임 계약

소년 전용 디자인 참조는 기존 두 소녀 참조를 덮어쓰지 않고 다음 세 장으로 별도 보존한다.

- `checkered-vest-boy-flat-sticker-action-sheet.png`
- `checkered-vest-boy-storybook-paper-action-sheet.png`
- `checkered-vest-boy-soft-3d-toy-action-sheet.png`

각 스타일은 기존과 동일하게 `run-jump`, `slide-fall` 4×4 원화 시트 두 장, 액션별 실제
8프레임, `256×256` RGBA, 발 피벗 `(128, 232)`를 사용한다. 슬라이드는 2프레임 진입,
4프레임 반복 유지, 2프레임 회복을 유지한다. manifest는 source를 인물군과 스타일로 구분하는
v4로 올리고, 최종 인벤토리는 디자인 6장, 프레임 시트 18장, 정규화 PNG 288장, 액션 GIF
36개, 슬라이드 단계 GIF 27개다.

런타임은 `RUNNER_CHARACTERS` 데이터로 선택 UI와 액션 파일을 연결하므로 물리·상태 코드는
바꾸지 않는다. 현재 목록은 `pink-glasses-girl-soft-3d-toy`와
`checkered-vest-boy-soft-3d-toy` 두 항목만 제공한다. 선택 UI 표시 이름은 각각 `이엘이`,
`이안이`이며 기본 캐릭터는 이엘이다.
localStorage 키는 유지한다. 구형 분홍안경·소년 스타일 ID는 같은 인물의 소프트 3D 버전으로
이관하고, 꽃모자 또는 알 수 없는 ID는 기본 분홍안경 소녀로 정규화한다.

### 검증

최종 생성·정적 검증·빌드·Playwright 결과는
`docs/endless-runner-character-animation-handoff-2026-07-15.md`에 기록한다.

생성 자산 검증은 9캐릭터, 디자인 6장, 프레임 시트 18장, 정규화 PNG 288장, 액션 GIF
36개, 슬라이드 단계 GIF 27개 계약으로 통과했다. 2종 런타임 축소 후에는 Vite 빌드
(111 modules), 실제 번들 22개 캐릭터 파일과 제외 자산 0개, 무한 러너 Playwright 5/5를
다시 확인했다. 전체 렌더 회귀 103/103은 2종 축소 직전 결과이며 이번 변경에서는 전체
스위트를 재실행하지 않았다.

2026-07-21 소프트 3D 런타임 승격 후 정적 자산 계약, Vite 빌드(195 modules), 소프트 3D
번들 22개 및 플랫 런타임 파일 제외, 무한 러너 Playwright 6/6을 다시 확인했다.

같은 날 실제 플레이에서 화풍 차이가 작게 보인다는 피드백을 반영해 프로덕션 캐릭터 2종의
4×4 원화 시트 4장을 프리미엄 3D 렌더로 다시 제작했다. 특히 발목과 신발 앞코·밑창·좌우 발
경계를 별도로 보정하고, 게임 표시 크기는 모바일 `76px → 92px`, 최대 `94px → 112px`, 선택
미리보기는 모바일 `48px → 62px`로 확대해 옷감·머리카락·신발 질감이 실제 화면에서도 읽히게
했다. 물리 충돌 상자와 액션 타이밍은 변경하지 않는다.

## 초원 지형과 장애물 확장 (2026-07-21)

바닥이 화면을 과도하게 차지하고 장애물 종류가 단조롭다는 피드백을 반영해 지면 기준을
`72% → 82%`로 내렸다. 잔디·흙 지층·자갈·꽃, 구름·원경 언덕·장식용 공중섬을 패럴랙스로
추가하고 그루터기·이끼 바위·가시밭·공중 잔디 지형·벌·파랑새를 캐릭터와 같은 소프트 3D
화풍의 런타임 에셋으로 연결했다. 라운드별 해금, 충돌 판정과 코인 안전 동선의 상세 계약은
`docs/endless-runner-terrain-obstacles-2026-07-21.md`에 기록한다.
공중 잔디 지형은 비행 장애물과 달리 위에서 착지할 수 있고 옆면·아랫면 접촉은 안전한 발판으로
분리한다. 발판이 캐릭터 아래에서 지나간 뒤에는 중력 낙하 상태로 자연스럽게 복귀한다.
벌과 파랑새는 월드 스크롤 속도에 생물별 접근 속도를 더해 오른쪽에서 플레이어 쪽으로 직접
날아오며, 진행 반대 방향의 비행 잔상으로 정적인 공중 지형과 움직임을 구분한다.
