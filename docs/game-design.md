# 달려라 검단호수공원 호반써밋 게임 설계

## 목표

`달려라 검단호수공원 호반써밋`은 최대 800명의 참가자를 조별로 경주시키고, 운영자가 지정한 순위 범위 안에 들어온 참가자를 통과자로 뽑는 웹 기반 경마 추첨 게임이다.

## 핵심 규칙

- 참가자는 1명부터 800명까지 입력할 수 있다.
- 경기는 시뮬레이션 시간 기준 84초 동안 진행된다.
- 트랙은 3가지다.
  - 호수 리본 코스: 안정적인 기본 트랙
  - 언덕 스퍼트 코스: 오르막 감속 뒤 내리막 스퍼트
  - 숲길 절벽 코스: 완만한 숲길과 절벽 징검다리 연출
- 장애물은 현재 비활성화한다.
- 조별로 헬기가 3회 등장하며 사격 순간까지 타격 대상 이름을 숨긴다.
- 참가자별로 5% 확률 기반의 스퍼트 스킬이 한 번 발동하고, 최소 2명은 스킬을 가진다.
- 스킬은 발동 후 10초 동안 속도를 높인다.
- 최종 84초 시점의 순위가 지정 범위 안이면 통과한다.

## 화면 구성

- 왼쪽 운영 패널: 참가자 입력, 샘플 생성, 트랙 선택, 통과 순위 범위, 시드, 재생 속도, 실행 버튼
- 중앙 경기장: 현재 시간, 헬기 이벤트 진행, 3D 트랙, 상위권 레이스 라인, 실시간 순위
- 오른쪽 결과 패널: 통과자 수, 스킬 발동 수, 헬기 탈락 수, 통과자 명단, 전체 순위 CSV 내보내기

## UI 개선 방향

- 레이스 화면은 호수, 나무 라인, 관중석, 출발선, 결승선이 있는 경기장 형태로 구성한다.
- 레이스 화면은 React Three Fiber 기반 3D 경기장으로 구성해 FHD 운영 화면에서 스크롤을 최소화한다.
- 헬기 이벤트는 별도 이벤트 모델로 관리해 추후 다른 연출을 추가하기 쉽게 한다.
- 움직이는 참가자 표시는 이름을 왼쪽, 말 이미지를 오른쪽에 배치해 선두 위치를 더 쉽게 비교하게 한다.
- 스킬 발동과 헬기 탈락 상태는 색상과 컷신으로 구분해 운영자가 레이스 흐름을 빠르게 읽을 수 있게 한다.
- 참가자별 색상을 배정해 순위가 바뀌어도 같은 참가자를 쉽게 따라갈 수 있게 한다.
- 설정과 결과는 좌우 고정 영역 대신 레이어 패널로 열어 트랙을 가로로 길게 보여준다.
- 시드는 화면에서 숨기고 내부 재현성 값으로만 유지한다.
- 상단 조작 영역에는 설정, 결과, 규칙 갱신, 레이스 시작, 새 레이스, CSV 버튼을 둔다.

## 구현 구조

- `src/raceEngine.js`: 순수 게임 규칙과 시뮬레이션 로직
- `src/app.js`: 브라우저 UI 상태, 이벤트, 애니메이션, 결과 렌더링
- `styles.css`: 반응형 레이아웃과 레이스 화면
- `tests/raceEngine.test.js`: 핵심 규칙 검증

## 운영 메모

- 동일한 참가자 목록과 같은 시드를 사용하면 같은 결과를 재현할 수 있다.
- 시드를 비우면 실행 시점 기준으로 새 시드가 생성된다.
- 화면 재생 속도는 진행 애니메이션 속도만 바꾸며, 카메라 줌인과 헬기 컷신 시간은 실제 시간 기준으로 유지한다.
